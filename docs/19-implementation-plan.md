## Product & module naming

Product/codename: **SlopDesk** (remote workspace — matches the `slopdesk` repo). Module prefix `SlopDesk`.

```
slopdesk/
├── Package.swift                 # SPM: libs + CLI execs + tests (HEADLESS-buildable & testable)
├── Sources/
│   ├── SlopDeskProtocol/            # wire format: framing, MessageType, seq(int64), Hello/Ack — pure Swift, 0 platform dep
│   ├── SlopDeskTransport/           # NWConnection + TCP_NODELAY, dual data/control channel,
│   │                             #   BackedWriter/BackedReader (ET replay 64MB/4MB gate), reconnect handshake
│   ├── SlopDeskHost/                # macOS: PTY (openpty + posix_spawn createSession), session mgr,
│   │                             #   no-buffer PTY↔transport relay (QoS user-interactive), TIOCSWINSZ
│   ├── SlopDeskClient/              # shared client: connection mgr, reconnect, input encoding
│   └── SlopDeskTerminal/            # TerminalSurface protocol + HeadlessSurface (libghostty impl = GUI app target)
│   ├── slopdesk-hostd/              # exec: headless host daemon (PTY + transport)  ← buildable tonight
│   └── slopdesk-client/             # exec: headless CLI test client                ← buildable tonight
├── Tests/{SlopDeskProtocol,SlopDeskTransport,SlopDeskHost,SlopDeskClient}Tests/
├── Apps/                         # Xcode projects (GUI; depend on SPM + libghostty XCFramework) — scaffold
│   ├── HostApp-macOS/  ClientApp-macOS/  ClientApp-iOS/
├── ThirdParty/ghostty/           # build script + External.zig patch → libghostty.xcframework
└── docs/
```

**Why headless-first:** the whole PATH 1 byte pipeline (host PTY ↔ TCP/`TCP_NODELAY` ↔ client + replay-buffer reconnect) is the de-risked core, fully buildable + unit/integration-testable with **no GUI, no libghostty**. The renderer (libghostty surface in a Metal view) is part of the GUI app target, behind `TerminalSurface`; its XCFramework build is the only fragile step (needs Zig) and must never block the core.
