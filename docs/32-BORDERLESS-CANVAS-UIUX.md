## What changed

| Piece | What |
|---|---|
| Borderless panes | `CanvasItemView` drops `PaneChromeView`: bare `PaneLeafView` + rounded clip (6pt) + a **static shadow plate** (`.background` rect carries the shadow — never `.shadow` on the live terminal layer, which adds an offscreen pass to every 60 fps repaint). Focus cue = stronger shadow + accent pill glyph (no ring border). `PaneChromeView` survives **only** for the iOS-compact carousel. |
| Floating pill (`FloatingPaneHandle`) | Material capsule at pane **top-centre** (avoids the shell prompt at top-left + all 4 corner grips; iPadOS-pill idiom). Kind glyph (accent when focused) + shared `PaneStatusDot`. In trouble states the SAME pill **expands** to status text + live "retrying in Ns" countdown; connected-but-laggy (>100 ms) expands to RTT. Hit fill = `ScrollPanForwarder` (scroll over pill pans the canvas — BUG-2 class). |
| Drag-vs-click | ONE `DragGesture(minimumDistance: 0)` with a **dead-zone latch** (4pt macOS / 10pt iOS): below the zone nothing previews and mouse-up = CLICK (focus + menu popover); past it the latch holds for the whole gesture (returning to start pixel still commits a move, never opens the menu). Deliberately NOT SwiftUI `Menu` (kills drag on mouseDown) and NOT `NSMenu` (tracking runloop stalls video panes). |
| Menu (`PaneMenuView`) | Popover: header (glyph + live title + status dot + status + RTT + running…), Reconnect (when it can help), Maximize/Restore, Center in View, Rename…, New Pane (3 inline kind buttons; cascades off this pane, inherits its group), Group (DisclosureGroup: assign/checkmark/new/remove), Snap-to-Panes / Snap-to-Grid / Show-Grid toggles (`@AppStorage`, also the iOS snap-disable path), Close Pane (destructive). Rows dismiss the popover FIRST, then mutate on the next runloop turn (popover-anchor teardown safety). |
| Dead scrim (`PaneDeadScrim`) | `.unreachable`/`.failed` ONLY (never transient states → cannot flicker): dims the stale body, whole pane = "click to reconnect". Scroll still pans (SPF fill). |
| Smart snap (`CanvasSnap`) | Pure solver beside `CanvasGeometry`. Candidate classes per axis: **gutter** (16pt = Tidy gutter = group padding) > **edge** (min/max any-to-any, incl. flush butting) > **center** (mid↔mid) > **viewportEdge** (16pt-inset edges + centreline, recomputed per frame). Grid (quantum **16pt**) is FALLBACK only — objects beat the grid. **Hysteresis**: engage 8 / release 12 (grid 6/9) via per-axis `Stick` tokens threaded through `previous`; solver is otherwise a pure function of RAW translation → zero drift, breakaway lands under the pointer, correction ≤ release by construction. Resize: moving edges only, no centres, min-size-violating candidates **discarded** (never clamped — a guide can never lie). ⌘-drag disables everything (polled per solve). |
| Preview ≡ commit | `@GestureState` holds the full snapped `Resolution` (auto-reset on cancel → no stuck preview/guides); a plain `@State` chain mirrors the hysteresis token into `.onEnded` — both sequences see chain_{n−1} per event, and the solver is idempotent at fixed input, so commit equals the last preview even inside the hysteresis band. ONE `movePane`/`resizePane` per gesture (commit-on-end discipline intact). |
| Guides (`SnapGuideOverlay`) | Item-local overlay on the DRAGGED pane (local = canvas − previewOrigin; dragged pane is focused → zIndex 1M → lines render above siblings; nothing clips them). 1pt accent lines, +16pt overshoot; centre guides dashed `[4,3]`. Grid snaps draw NO line (dots are the affordance). Guides live in gesture state → a cancelled gesture cleans them up by construction. |
| Dot grid (`CanvasGridLayer`) | 2pt dots every **32pt** of canvas space (= 2× snap quantum, so every dot is an honest snap site; gutter-tiled panes sit half a cell apart). ONE over-sized `SwiftUI.Canvas` tile whose content depends only on size+colourScheme; per-frame pan changes only a pixel-rounded `offset mod 32` (GPU transform — no redraw, no shimmer). `.primary` at 0.07 light / 0.11 dark. Hidden under maximize + Show Grid toggle. Not an NSView → background `PanView` keeps every scroll/drag. |

## Guardrails held

- Identity: chrome swap is **compile-time** (canvas never references `PaneChromeView`); `.id(PaneID)`, the single always-mounted ForEach, and geometry-only maximize untouched. Verified live: maximize→restore cycles, zero glyph garble.
- Terminal body keeps raw `mouseDown` (gestures only on pill + grips); video panes focus via `RemotePaneContext.onActivate`, terminals via `onRequestFocus`.
- No per-frame store mutation: previews are view-state; guides are gesture-state; `raisePane` fires once per gesture.
- Pan-only camera; persisted frames stay free `CGRect`s (snapping is interaction-time only) — no schema change.

## Verification

- `CanvasSnapTests` 28 pure tests (engage/release boundaries, held-stick vs nearer newcomer, idempotence, correction bound, tie-break gutter>edge order-independence, viewport edge+centre, grid fallback + objects-beat-grid, resize pin/discard/gutter/hysteresis, guides span/kind, sanitize).
- Full suite **1789/0**; `check-ios.sh` PASS (iOS triple); `check-macos.sh --connect` PASS (E2E connect + AUTOTYPE type→exec→render with the new UI).
- Live cua-driver session (loopback rig): pill menu opens (all sections render, live RTT in header), New-Pane-from-menu works, pill drag moves the pane, **snap pixel-verified**: second pane dragged near the first → left edges land at x=352 EXACTLY both, vertical gap EXACTLY 16px (gutter class); grid snap observed (y → 384 = 24×16). Maximize→Restore via menu: panes return to exact frames, glyphs clean.
- Adversarial review workflow over the diff (5 dimensions × adversarial verify; 34 agents): **21 confirmed → 20 fixed**, 8 rejected. Fixed set, grouped:
  - **Perf (HIGH)**: `snapTargets`/`snapViewport` were per-item BODY inputs → every pane re-evaluated every pan frame (BUG-1/BUG-2 class). Now `snapEnvironment()` reads the store INSIDE the gesture closures at solve time (no observation dependency); `CanvasItemView` takes only frame-stable `viewportSize`. Grid tile split into `Equatable` `DotGridTile` so the draw closure provably never re-runs during a pan.
  - **Focus (HIGH)**: the popover's `.onDisappear { store.focus(id) }` re-raised the menu's pane on dismissal, stealing focus/z from whatever the user clicked next (best in-diff candidate for the observed first-drag misroute) — deleted; the main window's first responder survives a popover stint on its own.
  - **Maximize correctness**: a maximize toggle does NOT cancel an in-flight gesture (identity survives by design) — every move/resize closure now guards `displaySize`, a frozen preview is cleared in `.updating`, guides overlay gated off while maximized, and a maximized resize can no longer commit onto the hidden restore frame. A drag on a maximized pill is a no-op (only a true click opens the menu).
  - **Solver math**: near-tie selection rewritten as order-independent two-pass (`selectBest`) — the single-pass comparator was non-transitive, letting the winner depend on `others` order; a held stick now requires its source candidate to still EXIST (a closed/moved neighbour drops the hold — snap active ⇔ guide drawable, both ways); guides group by dragged-edge and draw at the EXACT snapped coordinate (no 0.5pt bucket offset/duplicates); the `.onEnded` commit replays the exact `Config` the last preview used (a ⌘ release between the last drag event and mouse-up can no longer flip preview ≢ commit).
  - **Hit-testing**: the dead scrim no longer swallows resize grips (declared before them — a dead pane can be resized; its interior stays the reconnect target). Stale chain mirrors cleared on gesture auto-reset (a cancelled drag could otherwise turn the next pill click into a zero-distance "move").
  - **iOS/a11y**: pill is ONE accessibility element with live status/latency/running folded into its label and an activate action (VoiceOver can open the menu); 44pt iOS hit target (12pt vertical padding vs 4pt macOS); "Tap to reconnect" copy on iOS.
  - **Information parity**: pill now shows a quiet truncated TITLE (always on iOS; hover/focus on macOS), a "running…" expansion + full opacity while a command runs, and the >100ms RTT badge is latched with a 110/90 hysteresis band (no width-pop at the boundary). Rename hosted IN the popover (inline TextField → `store.updateSpec`), so it no longer silently no-ops with the sidebar hidden. Dead scrim suppresses the in-leaf orange failure banner (reason stated once; the neutral "Session ended" banner and carousel untouched). Snap/grid toggles surfaced app-globally in the View menu (same `@AppStorage` keys); ⌘-bypass advertised in pill/toggle help.
  - Dead `moveHandleGesture` plumbing removed from `PaneChromeView` (carousel-only now).
  - **Deliberately NOT applied** (1 of 21): re-adding a 0.5pt hairline border for light-mode separation — the directive is explicitly "no border"; shadow + grid dots carry separation.
- Post-fix gates re-run: full suite **1792/0** (31 snap tests), `check-ios.sh` PASS, `check-macos.sh --connect` PASS.
