# E21 — Remote-window extension first-class through clone surfaces — IMPLEMENTATION PLAN

> **Framing (per carry-overs §0/§6): this is an AUDIT → FILL-GENUINE-GAPS epic, NOT build-from-scratch.**
> `.remoteGUI` (a real host window streamed over the UDP video path) must be a first-class peer of a
> terminal pane in *every* clone surface E1–E20 shipped. There is **no otty analog / no screenshot**; the
> standard is the existing aislopdesk surfaces. Most machinery already exists and is kind-generic — the
> diff is small and surgical EXCEPT one genuine net-new render piece (the floating-pane view).
>
> **Wire:** `touchesWire = false` (no golden change). **iOS:** `touchesIOS = true` (shared `AislopdeskClientUI`
> + `AislopdeskWorkspaceCore` — gate MUST run `scripts/check-ios.sh`). **Float-codec/controller:** `false`
> (the only float math is `CGRect` clamp geometry that already lives in `WorkspaceTreeOps`, pinned by tests;
> any new clamp reuses it and stays ordered-compare — no `fma`/codec).

## 1. Hard exclusions (carry-overs §4 — DO NOT build)

- **Per-host badge / multi-host selector in pickers** — no live multi-host today (`DECISIONS.md:123` keeps per-session multi-host schema-reserved). Document as a deferred extension point; ship **no** inert multi-host chrome.
- **Drop-to-create a remote-window pane** (file/URL drop) — out of scope; remote windows come from the picker. State it in `DropActionResolver` doc + a test asserting no `.remoteGUI` drop arm exists.
- **True PiP / per-pane OS float window** — deferred at E19. E21 "floating" = the in-app `FloatingPaneCard` in the split container, NOT an OS child window.
- **Horizontal/top tab bar + SSH-host filter** — globally dropped; do not reintroduce.
- **No new over-wire protocol.** Read-only on the video path is enforced **client-side** by *not forwarding* input (wire-compatible silence) — no VideoControl change, no golden touch.

## 4. Approach + verification per acceptance story

### ES-E21-1 — picker/connect overlay open and create `.remoteGUI` panes — GUI-verifiable
Audit only: `OverlayHostView` already mounts `ConnectHostView` + `RemoteWindowPickerModal` via E2's `OverlayCoordinator`; `WorkspaceStore+RemoteWindow.newRemoteWindowTab(windowID:title:appName:)` mints a `.remoteGUI` `PaneSpec`. **Verify** end-to-end: a pick → a `.remoteGUI` spec in the tree. **Test:** headless `RemoteGUIFirstClassPeerTests` asserting `newRemoteWindowTab` returns a `PaneID` whose `spec.kind == .remoteGUI` and is reachable in `tree`/`openedItems`. **GUI:** `scripts/check-macos.sh` on a real session (open picker → pick window → pane streams) — code-review the render path (hang-safe, never unit-instantiated).

### ES-E21-2 — peer in palette/Open-Quickly, sidebar (status), status bar, drag-drop, zoom, read-only — both
- **Open-Quickly (WI-2):** confirm `openedItems` already includes the video pane; add differentiation. **Test:** build a `TreeWorkspace` with a `.terminal` + a `.remoteGUI` pane, assert both appear and the video row carries the window glyph/badge/subtitle. Revert-to-confirm-fail by removing the `paneKind` thread.
- **Read-only on video input (WI-3):** **the load-bearing correctness item.** Pure policy: `store.isReadOnly(for:)` already drives the `paneReadOnly` set kind-generically; `GuiLeafView` translates it into `RemotePaneContext.inputEnabled`. **Test:** `setPaneReadOnly(videoPane, true)` ⇒ `isReadOnly(for:) == true`; and a seam test asserting the context built for a read-only video pane carries `inputEnabled == false` (and the keyInjector sink is cleared). The actual CGEvent suppression is app-target (`AislopdeskVideoClient`), gated on `isActive && inputEnabled` — code-review only (hang-safety). **Also** verify the pill + `OttyTabRow.readOnly` lock show on a read-only `.remoteGUI` pane (already kind-generic).
- **Status bar (WI-4):** mount `StatusBarStrip` in `GuiLeafView`. **Test:** `StatusBarContent.make(kind: .remoteGUI, …)` yields `paneKind == "remote"`, `exit == .none`, empty cwd (existing `StatusBarModelTests` extended).
- **Sidebar (WI-5):** **Test:** `RailRowsBuilder.rows(...)` for a `.remoteGUI` pane → a row with the window subtitle and the readOnly lock when locked; connection-dot folded if shipped.
- **Drag-drop (WI-7):** **Test:** split a `.remoteGUI` pane's tab (kind-generic `WorkspaceTreeOps.split`) → both leaves present; a foreign drop bundle actuated against a `.remoteGUI` target (`terminalModel == nil`) is a no-op (no crash).
- **Zoom:** **Test:** `Tab.zoomedPane = videoPaneID` → `SplitTreeRenderModel.layout` returns one full-bounds leaf, no floats — kind-agnostic.
- **GUI:** `scripts/check-macos.sh` — focus a `.remoteGUI` pane, confirm sidebar row, status bar, zoom, and that read-only blocks clicks/keys.

### ES-E21-3 — float a remote-window pane as a draggable/resizable card — GUI-verifiable
Build `FloatingPaneCard` + wire `floatingLeaves` (WI-6). The float DOMAIN (`toggleFloating`/`moveFloating`/`resizeFloating`/`raiseFloating`/clamp) is already tested; the NEW surface is the view. Kind-generic `PaneContainer` means `.remoteGUI` floats for free. **Test (pure):** extend `SplitTreeRenderModelFloatingTests` — a tab with a floating `.remoteGUI` pane emits a clamped `floatingLeaf` at its `floatingFrame` (z-ordered); `WorkspaceFloatingPaneTests` confirm `toggleFloating` floats a `.remoteGUI` active pane (no kind guard). **GUI:** `scripts/check-macos.sh` — ⌥⌘F floats the focused remote window → a draggable/resizable card; drag/resize commit once on release; embed/close return it to the tile. Verify no surface teardown (the video surface keeps streaming across float/embed).

### ES-E21-4 — every surface treats `.remoteGUI` as a first-class peer (no special-casing that drops it) — unit-testable
`RemoteGUIFirstClassPeerTests` (the audit-as-tests) + a final grep sweep for enumeration sites that drop `.remoteGUI` (a `switch` over `PaneKind` with a `.terminal`/`.web` arm + a `default` no-op, or a `case .terminal:` guard that silently excludes). **Test:** assert the model entry points (`openedItems`, `paneKindLabel`, `RailRowsBuilder.rows`, `toggleFloating`, `split`, `isReadOnly`) all admit/handle `.remoteGUI`. **Sweep:** `grep -rn "case .terminal" Sources | grep -v remoteGUI` and review each for an unintended drop.
