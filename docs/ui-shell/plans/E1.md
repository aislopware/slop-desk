### The TWO genuine remaining gaps (the actual work)

1. **N5 — named / parameterized config actions are PARSED but DROPPED end-to-end.**
   `KeybindGrammar.parseAction` returns `.named(id:arg:)` for `new_tab`, `goto_tab:1`, `split_right`, …,
   and `KeybindConfigLoader.apply` routes them through an OPTIONAL `resolveNamedBinding` hook — but the
   launch call passes **no resolver** (`SlopDeskClientApp.swift:85`:
   `KeybindConfigLoader.loadFile(at: configURL, into: preferences.keybindings)` — no `resolveNamedBinding:`),
   and **no action-id name table exists** mapping the spec's config action names (`new_tab`, `split_right`,
   `goto_tab:N`, …) to registry binding ids (`tab.new`, `pane.splitRight`, `tab.select.N`). So a user's
   `keybind = cmd+t:new_tab` silently does nothing. The hook is unit-tested with a fake resolver
   (`KeybindConfigLoaderTests:126`); the **production resolver** is the missing piece. ES-E1-6's literal
   half works; the BACKLOG's "param-action parse (N5)" is unwired.

2. **N6 — SwiftUI `.commands` menu-bar absent (OPTIONAL in the BACKLOG).** No `WorkspaceCommands: Commands`,
   no `.commands { }` on the `WindowGroup`. Every workspace chord fires through the `NSEvent` monitor
   (`WorkspaceKeyDispatcher`), which is correct and load-bearing (a multi-key prefix can't be a
   `.keyboardShortcut`, and a `.commands` shortcut can't swallow a prefix follow-up before libghostty —
   see the dispatcher header). N6 is a discoverability nicety. **Recommended: a thin, shortcut-less menu**
   (titles + click → `route`, NO `.keyboardShortcut`) over the existing single source of truth.

## Flags (CLAUDE.md compliance)

- **Wire protocol / golden corpus:** **NOT touched.** No new wire message / codec change / golden key.
  The `text:`/`csi:`/`esc:` bytes ride the EXISTING PTY mux `sendBytes` path. `KeybindGrammar` /
  `KeybindConfigLoader` / the new name table are pure String→struct transforms. `make golden` must stay a
  no-op diff — run it to PROVE no byte moved. `touchesWire = false`.
- **Float-math / codec / controller:** **NOT touched.** No FMA, no float compares, no FEC/codec.
  `touchesFloatCodec = false`.
- **iOS (`#if os(iOS)`):** The dispatcher + the menu are `#if os(macOS)`. The registry, routing,
  `KeybindGrammar`, `KeybindConfigLoader`, `KeybindingPreferences`, and the NEW name table (WI-1) are
  platform-neutral and **DO compile into the iOS slice** (they live in `SlopDeskWorkspaceCore` /
  `SlopDeskVideoProtocol`). iOS rots silently — **run `bash scripts/check-ios.sh` after WI-1/WI-2**
  even though no new code sits under `#if os(iOS)`. The menu (WI-4) is macOS-only, adds no iOS surface.
  `touchesIOS = true` (precaution: shared modules compile on iOS).
- **Validate-then-drop on untrusted input:** config text is untrusted. `KeybindGrammar` /
  `KeybindConfigLoader` already DROP a malformed line and never trap (no `!`, bounded `\xNN`). The new
  name table (WI-1) must likewise return `nil` for an unknown id / out-of-range arg (drop, never crash),
  and the resolver must not force-unwrap.
- **No app-layer crypto / no version negotiation:** unaffected.

## Traps specific to E1

- **Do NOT add `.keyboardShortcut` to the N6 menu items.** The dispatcher header is explicit: a
  `.commands` shortcut can't swallow a multi-key prefix follow-up before libghostty, and would double-fire
  alongside the NSEvent monitor. Menu items are discoverability only.
- **`copy_to_clipboard` / `paste_from_clipboard` / `select_all` have NO `WorkspaceAction`** — they are
  libghostty's own responder actions (`TerminalContextMenu.swift`). The name table returns `nil` for them
  (drop), NOT an invented id. The config example `ctrl+shift+c:copy_to_clipboard` is a libghostty
  binding, out of E1 scope, correctly dropped.
- **Layering:** keep the registry import out of `SlopDeskVideoProtocol`. The resolver crosses the
  boundary only at the app layer (WI-2). This is why the hook exists.
- **iOS rot:** WI-1 compiles into the iOS slice; macOS `swift build` won't catch an iOS type error in it —
  `scripts/check-ios.sh` is mandatory.
- **`reopenLastClosedPane()` is an intentional empty stub** (E3 fills the LIFO). Do not "fix" it in E1 —
  the route is live, the behavior is delegated; `testE1StubActionsDoNotMutateTree` pins it a no-op.
