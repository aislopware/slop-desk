# E11 — Open Quickly (`⌘⇧O`) + per-item Actions popover — implementation plan

**Goal.** Xcode-style `⌘⇧O` multi-source quick switcher, reusing E2 palette+fuzzy infra and the E10 Jump-To stack. One floating picker, one search field, fuzzy across **Opened / Recent / Folders / Agents / Current** plus a merged **All**; `⌘K` Actions popover per item; `⌘1–9` direct quick-pick; `Tab`/`⇧Tab` pill cycling.

**Acceptance:** ES-E11-1 … ES-E11-4 (USER-STORIES.md). Carry-overs in `plans/E11-carryovers.md` folded in below as hard constraints.

## Binding constraints (from carry-overs — REQUIRED)

- **NO SSH pill / no `~/.ssh/config` parse / no `⌘S` picker chord / no "SSH host" Actions row.** Pill set = **All / Opened / Recent / Folders / Agents / Current** (SSH + Recipes dropped).
- **OMIT the Recipes pill** (no backing engine until E16). Leave a code note at the pill array: `// Recipes pill: added in E16 once the recipe store exists`.
- **Agents = Claude Code ONLY.** Filter `listAgentSessions` to `agentKind == .claude`.
- **Vertical-rail model only** — "Opened" enumerates `tree.sessions[].tabs[].root` panes; no horizontal tab-bar concept.
- **Reuse, do not rebuild the picker.** `⌘⇧O` action + `OverlayCoordinator.openQuickly` case + `WorkspaceBindingRegistry`/`Routing` stubs exist (live but routed to `break`). Fill them.
- **Per-filter chords (`⌘0/⌘W/⌘R/⌘Z/⌘G/⌘J`), `⌘1–9`, `Tab`/`⇧Tab`, `⌘K` are PICKER-LOCAL** — handled by the panel's `onKeyPress`, NEVER in `WorkspaceBindingRegistry`. Only `⌘⇧O` (open All) and `⌘J` (open Current) are global. `⌘J` is already owned by E10 — reuse, don't double-bind.
- **Reveal/Open route to the HOST via E10's `MetadataVerb.openPath = 9` / `revealPath = 10`.** Copy actions = client pasteboard. Change Directory Here = **verbatim UTF-8 `cd <quoted>\n`** into the PTY (cd to PARENT if target is a file). Re-Run = existing verbatim re-run path.
- **NO new wire. Golden stays ZERO-DIFF.** No float/codec/controller code. Shared `SlopDeskClientUI` must compile for iOS — run `scripts/check-ios.sh` in the gate.

## Architecture decision (Open Quickly / Jump-To reconciliation)

Open Quickly and Jump-To are the SAME picker opened to different default pills. Adopt **carry-over option (a): fold the E10 Jump-To panel into the new Open Quickly picker.**

- ONE picker view `OpenQuicklyView` (ClientUI) replaces the standalone `JumpToView` mount.
- ONE pure model `OpenQuicklyModel` (WorkspaceCore) with an `OpenQuicklyFilter` enum (`all/opened/recent/folders/agents/current`) — the Open-Quickly pill taxonomy, distinct from the palette's `QueryFilter` (the `⌘⇧P` command-palette concept, NOT reused for pills).
- The **Current** filter's data source IS the existing `JumpToModel` (reused verbatim — headless tests stay green; `JumpToItem`/`filtered`/`itemKind` unchanged).
- `⌘J` re-points to "open OpenQuickly at `.current`"; `⌘⇧O` opens at `.all`. The E10 `jumpToVisible` flag + `JumpToView.swift` are removed; their link-action actuation is extracted to a reusable `LinkActionActuator` so Current rows + File/Folder rows share one actuator (no regression to `LinkActionPolicy`/`TerminalContextMenu` routing).

Why a new model rather than `PaletteMode.openQuickly`/`QueryFilter`: the palette's `QueryFilter` (Actions/Tabs/Sessions/Files/Conversations/Repos) is the ⌘⇧P taxonomy and doesn't match the Open-Quickly pills. `OpenQuicklyFilter` keeps the two surfaces honest and lets All merge the six real sources with section headers. `PaletteMode.openQuickly` + `multiSource` become dead and are removed (or left as the ⌘⇧P-only path — see WI-7 note); `EmptyPaletteSource(.files/.conversations/.repos)` stubs are no longer surfaced by E11 (never reachable).

## Per-story approach + verification

### ES-E11-1 — `⌘⇧O` opens the switcher with pills, default All
- **Approach.** Fill `WorkspaceBindingRouting case .openQuickly` → `toggles.openQuickly?()`; app binds to `overlay.toggleOpenQuickly(filter: .all)`. `OpenQuicklyView` renders the pill bar **All/Opened/Recent/Folders/Agents/Current** (SSH+Recipes omitted, code-note at the array). Active pill = filled (`Slate.State.selected`), inactive = outlined. Default `openQuicklyFilter = .all`.
- **Verify.** GUI (`check-macos.sh` autoconnect + cua screenshot vs `screenshots/open-quickly.png`). Unit: `OpenQuicklyFilter.pickerPills` excludes SSH/Recipes; `OpenQuicklyModel` defaults to `.all`.

### ES-E11-2 — `Tab`/`⇧Tab` cycle filters; Opened lists every live pane; `↩` switches
- **Approach.** Picker-local `onKeyPress(.tab)` → `coordinator.setOpenQuicklyFilter(model.nextFilter(...))` (`⇧Tab` → prev). Opened source = enumerate `tree.sessions[].tabs[].root.allPaneIDs()` → one `OpenQuicklyItem(.pane)` per pane (title=lastKnownTitle/title, subtitle=lastKnownCwd, agent icon); fuzzy via `FuzzyMatcher`. `↩` runs the item's `Act` → `.focusPane(id)` → `store.focusPaneTree(id)` + close.
- **Verify.** Unit: `nextFilter/prevFilter` wrap order; Opened builder emits N rows for N panes; fuzzy narrows. GUI: Tab cycles pill highlight; ↩ on a pane focuses it.

### ES-E11-3 — `⌘K` Actions popover (context actions) + `⌘1–9` direct open
- **Approach.** Reuse `JumpToView.actionsPopover` shape. Per-kind action tables (Pane/Folder/Agent/Command/File-URL) — **SSH row dropped**. Picker-local `onKeyPress("k", .command)` toggles the popover on the selected row. `onKeyPress("1"…"9", .command)` → `model.quickPickIndex` → run the Nth VISIBLE selectable row directly. Actions actuate via `LinkActionActuator` (Copy=pasteboard; Reveal/Open=host verbs 9/10; Change Directory Here=verbatim `cd`, parent-if-file; Re-Run=verbatim; Forget Folder=`folders.forget(path:)`; Close Tab/Move=store ops).
- **Verify.** Unit: `quickPickIndex(3, in:rows)` maps to the 3rd selectable; per-kind action table has no SSH row; Change-Directory builds `cd <parent>` for a file path (reuse `LinkActionPolicy` tests pattern). GUI: ⌘K shows the popover; ⌘2 opens the 2nd row.

### ES-E11-4 — Folders frecency + Agents (Claude, current project)
- **Approach.** Folders = `folders.ranked(now:)` → `OpenQuicklyItem(.folder)` rows (honest "No folders yet" empty-state until a cwd is visited). `WorkspaceStore.setLastKnownCwd` fires `onCwdVisited` → `folders.record(cwd:)` (bounded, validate-then-store). Agents = `await activeMetadataClient.listAgentSessions(project: activeCwd)` filtered to `agentKind == .claude` → `OpenQuicklyItem(.agent)` rows (title, project subtitle, mtime stamp); loaded async on appear + on switching to Agents/All, with an in-flight/empty state. iOS sources Agents from the host too (metadata RPC is host-served).
- **Verify.** Unit: `FolderFrecency.score` orders frequent×recent above stale; cap evicts oldest; `record` rejects an over-long path; persistence round-trips and decode-fails to empty. Agents builder drops non-Claude kinds. GUI: `cd` into dirs then ⌘⇧O→Folders shows them ranked; Agents lists Claude sessions (`screenshots/open-agents-quickly.png`).
