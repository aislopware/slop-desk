# E13 — Agent integration UI (Claude Code) — implementation plan

**Epic goal.** Surface the existing host agent-detection stack as slopdesk's complete agent experience:
Agents settings card (install/uninstall hooks + status), 7 behavior toggles (badge×3, notify×2,
prevent-sleep, resume-on-recovery) with per-pane overrides + Clear-Badge, the AgentInputFooter mount,
Send-to-Chat (`⌘⌃↩`), History viewer additions (raw-JSONL toggle + Resume), Fork (palette "Fork in…"),
and the Peek-and-Reply overlay (`⌘⌥J`). **Claude Code ONLY.**

**Binding constraints (from `plans/E13-carryovers.md`, both REQUIRED):**
1. **Claude only** — never surface `MetadataCodec.AgentKind.codex` in any UI (install card, badges,
   history, fork, send-to-chat picker). The enum case stays; it is documented-dead, never rendered.
2. **NEVER an approval gate** — supervision is observe + notify + reply-on-user-initiative. No flow may
   pause the agent pending an slopdesk confirmation. (E17 per-pane read-only input gate is unrelated.)

**Flags:** `touchesWire: true` (new metadata verbs 11/12/13 — full golden discipline) · `touchesIOS: true`
(shared `SlopDeskClientUI`; the gate MUST run `bash scripts/check-ios.sh`) · `touchesFloatCodec: false`
(no float-math / wire-codec / realtime-controller change; the new wire bytes are manual big-endian only).

This is the **most reuse-heavy epic** — almost every engine exists. Read the VERIFY-NOT-REBUILD map below
before writing a line.

## Wire posture (touchesWire: true) — extend E4's metadata channel

Add THREE verbs after `revealPath = 10`, following the `openPath`/`revealPath` discipline (status-byte +
empty/short payload, no exfil, `init(rawValue:)`→nil→`.unsupportedVerb`, validate-then-drop, ALWAYS reply):

| New verb | # | Direction | Request payload | Response | Side effect |
|---|---|---|---|---|---|
| `installAgentHooks` | 11 | client→host | empty | status byte, empty payload | host writes `~/.claude/settings.json` + hook script via `AgentInstaller.install` |
| `uninstallAgentHooks` | 12 | client→host | empty | status byte, empty payload | host strips our entries via `AgentInstaller.uninstall` |
| `agentHookStatus` | 13 | client→host | empty | status `.ok` + **1-byte** payload (`1` installed / `0` not) | NONE (pure read of the marker) |

**Prevent-sleep + resume-on-recovery do NOT need a wire verb** — they are host-LOCAL policies. Following
the existing `agentDetect`/`agentHooks` precedent, they ride the **`AgentPreferences` sidecar** (reconnect-
tagged) to `HostEnvironment`; the host holds the `IOPMAssertion` driven by the EXISTING `claudeStatus
.working` detection it already computes. This keeps those two toggles off the live wire (honest mapping:
they apply on the next host connection, exactly like the other host flags — surfaced with the
`.reconnect` timing chip). Install/uninstall/status DO need the live verbs (the user clicks Install and
expects an immediate file write + a status flip), so `touchesWire: true`.

**Golden discipline:** run `swift run slopdesk-corevectors` (NO `SLOPDESK_*` env) + `bash
scripts/golden-check.sh`; the **13 frozen keys must stay intact** and the emitted-key set byte-identical
EXCEPT any vector that round-trips a new verb (E10 added 9/10 with effectively zero diff — match that).
Hand-edit `golden/golden_vectors.json` surgically if a new vector is added; NEVER redirect the generator
over it. Update `docs/20-wire-protocol.md` (the MetadataVerb table) + `docs/DECISIONS.md` (E13 entry).
Host+client redeploy together (no backcompat; host accepts only version 1).

**Settings card ↔ wire seam.** The card is global, but `MetadataClient` is one-per-pane (pane identity =
mux channel). Install/uninstall act on the host regardless of pane, so the card routes through the FIRST
connected pane's `MetadataClient` (resolved via the app's connection registry). With no connected pane the
card disables the buttons and shows "Connect a session to manage hooks" (honest, never a dead button). A
new `AgentHooksController` (below) carries injected async seams so the model is unit-tested with a fake.
