  `NSItemProvider`/`NSPasteboard` specifics are `#if os(macOS)`-gated; the SwiftUI `DropDelegate`

- **touchesFloatCodec = false.** No FEC/controller/codec/float math. Zone hit-test uses ordinary

## Approach per acceptance story + verification

### ES-E18-1 ‚Äî drag a file over a pane shows the 5 zones; hovered zone highlights ‚Äî GUI-verifiable
- `PaneDropReceiver` (`DropDelegate`) on each `PaneContainer`. `dropEntered` classifies via
  `DropPayloadClassifier`; if `nil` (unsupported) ‚Üí `false`, no overlay. `dropUpdated(info:)` maps
  `info.location` through `PaneDropZoneLayout.zone(at:)` ‚Üí `@State activeZone`; `PaneDropOverlay` draws
  the 5 blobs and saturates the active one (warm-grey ‚Üí olive-green). Disabled zones (green-half for
  file/URL) render muted + non-targetable (`DropActionResolver.resolve == nil`).
- **Verify:** unit ‚Äî `PaneDropZoneLayoutTests` (zone geometry + `zone(at:)`), `DropPayloadClassifierTests`
  (type routing). GUI ‚Äî Phase-3 HW: `scripts/check-macos.sh` + cua drag a Finder file over a pane,
  screenshot overlay + active-zone tint vs `drop-overlay-frame-action.png` (live overlay render is
  headless-unprovable ‚Üí Phase-3 target, do not fake a pixel proof).

### ES-E18-2 ‚Äî Insert-Path injects the path; folder on New-Tab cd's (host-resolved note) ‚Äî both
- `performDrop` ‚Üí `DropActionResolver.resolve(zone:content:)` ‚Üí actuate:
  - `injectText(path)` ‚Üí `terminalModel.sendInput(Data(path.utf8))` VERBATIM (focused pane).
  - `newTabCd(folder)` ‚Üí `store.newTab(kind:.terminal)` then send
    `LinkActionPolicy.changeDirectoryCommandLine(folder)` (a dropped FILE cd's to its PARENT via that
    helper ‚Äî never bare `cd '<file>'`; NOT the ‚åòT `openChooserPane(.newTab)` path). THEN
    `overlayCoordinator.pushToast(Toast(id:"drop-cwd", flavor: .attention, title:"cd'd on host",
    body:"<path> is resolved on the host; it may not exist there."))` ‚Äî advisory (pane is local, its
    session is remote), never block.
  - `hostOpen(path)` (Open-In-Place) ‚Üí `await metadataClient.openPath(path)`; on `false` push an error
    toast (never a silent no-op, the E10 idiom).
- **Verify:** unit ‚Äî `DropActionResolverTests` (zone√ócontent table). The cd idiom is pinned by
  `LinkActionPolicyTests`; add a `WorkspaceStore`-level test that `newTabCd` sends `cd ‚Ä¶ || cd <parent>`
  (inject a fake terminal sink). GUI ‚Äî Phase-3 HW: drop a folder on New-Tab, confirm shell `cd`s + toast.

### ES-E18-3 ‚Äî drag a sidebar tab to reorder with an insertion-line indicator ‚Äî both
- Reorder itself DONE (E6). WI-7 adds the insertion-line via `.dropDestination(‚Ä¶, isTargeted:)`: a 2pt
  accent rule on the targeted row's top edge. By-uuid drag, search/grouping gating, foreign-drop
  rejection UNCHANGED.
- **Verify:** unit ‚Äî `TabDragPayloadTests` already pins `resolveMove` (by-uuid, mid-drag `.updated`
  reshuffle, OOB/self no-op) ‚Äî extend with a targeted-row resolution case. Insertion line is a pure
  overlay ‚Üí assert `isTargeted` flips a published flag (ViewInspector-free: test the small
  `DropInsertionIndicator` model if extracted). GUI ‚Äî Phase-3 HW: drag a rail tab, screenshot the line.

### ES-E18-4 ‚Äî open a URL in a built-in web pane (WKWebView, non-persistent store, no autoplay) ‚Äî GUI-verifiable
- URL drop / `splitWeb` / `openWeb` ‚Üí `store.openWebPane(url:placement:)` creates a `.web` leaf;
  `WebLeafView` renders address-bar chrome + `WebRendererFactory.make`. App target registers the factory
  at launch (like `VideoWindowFactory.shared`): production view builds a `WKWebView` with
  `configuration.websiteDataStore = .nonPersistent()` (D9, no on-disk cookies/cache),
  `mediaTypesRequiringUserActionForPlayback = .all` (no autoplay), `javaScriptEnabled` default. Headless
  library renders the placeholder. `WebURLNormalizer` maps bare host ‚Üí `https://`, free text ‚Üí DuckDuckGo,
  validate-then-drops a malformed/non-http(s) string.
- **HANG-SAFETY (binding):** WKWebView is GUI/WebKit ‚Üí never instantiated in a test or headless build;
  lives ONLY in the app target behind `WebRendererFactory`. Tests exercise only pure parts
  (`WebURLNormalizer`, `PaneKind.web` plumbing/persistence, drop geometry).
- **Verify:** unit ‚Äî `WebURLNormalizerTests`, `WebPaneStoreTests`, `PaneKindWebDecodeTests`. GUI ‚Äî
  Phase-3 HW: register the factory in the app target, drop a URL, screenshot the rendered page vs
  `web-broswer.png` (live WKWebView render headless-unprovable ‚Üí Phase-3 target).
