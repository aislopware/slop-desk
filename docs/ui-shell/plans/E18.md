## Verdict after reading the code (VERIFY-NOT-REBUILD)

- **ES-E18-3 (tab reorder) is ALREADY BUILT in E6 and acceptance-complete except the *insertion-line
  indicator*.** `NavigatorColumn.swift:103 handleTabDrop` + `:118 reorderable(_:row:)` wire
  `.draggable(TabDragPayload.encode(row.tabID))` + `.dropDestination(for: String.self)` →
  `WorkspaceStore.moveTabRendered(from:to:)` → `WorkspaceTreeOps+MoveTab.swift`. The drag carries tab
  IDENTITY (uuid), rejects foreign payloads, is gated during search/grouping. **Do NOT rebuild any of
  this.** The ONLY genuine gap vs the spec ("a thin insertion-line indicator shows the landing
  position between tabs") is a drop-target highlight — added as WI-7 (a `.dropDestination(…,
  isTargeted:)` top-edge insertion line), nothing more.
- **External drop overlay: genuinely NEW.** No `.onDrop` / `DropDelegate` / `NSItemProvider` /
  `NSDraggingDestination` exists anywhere (`grep` clean). Build the overlay + drop receiver new, but
  REUSE every actuator (split machinery, new-tab op, the single PTY input funnel, the cd-parent helper,
  the snippet expander, the E10 host openPath verb).
- **Web pane: genuinely NEW.** `PaneKind` (`PaneSpec.swift:48`) has terminal / remoteGUI / systemDialog
  / chooser — no `.web`; no `import WebKit` / `WKWebView` anywhere. Add `case web` + a headless-safe
  factory seam mirroring `VideoWindowFactory` / `TerminalRendererFactory`.

## Wire / iOS / float-math flags

- **touchesWire = false.** Every drop injects verbatim through the existing PTY funnel
  (`TerminalViewModel.sendInput`), reuses E10's `MetadataVerb.openPath = 9` / `revealPath = 10` for
  Open-In-Place, and the web pane is fully local. NO new message, NO golden-corpus change. The gate's
  `bash scripts/golden-check.sh` MUST show the 33 emitted keys byte-identical + 13 frozen intact (zero
  drift). If a host path-stat is ever wanted for the cwd warning, REUSE E4's metadata channel — do not
  add a message. (Default: no host stat; inject verbatim + advisory toast, matching this design's
  local-verbatim behavior.)
- **touchesIOS = true.** A new `PaneKind` case + the `WebRendererFactory` seam live in shared
  `SlopDeskWorkspaceCore` / `SlopDeskClientUI`; WKWebView exists on iOS (UIKit) too. The macOS
  `NSItemProvider`/`NSPasteboard` specifics are `#if os(macOS)`-gated; the SwiftUI `DropDelegate`
  (`.fileURL`/`.url`/`.text`) is cross-platform. **The gate MUST run `bash scripts/check-ios.sh`** —
  `swift build` on macOS will not type-check the `#if os(iOS)` slice.
- **touchesFloatCodec = false.** No FEC/controller/codec/float math. (The zone hit-test geometry uses
  ordinary `CGFloat` distance math — no `<`/`>` NaN traps on attacker input; inputs are local pane
  sizes, not untrusted UDP. Still: use plain comparisons only on finite local geometry, never fuse.)

## Approach per acceptance story + verification

### ES-E18-1 — drag a file over a pane shows the 5 zones; hovered zone highlights — GUI-verifiable
- `PaneDropReceiver` (`DropDelegate`) on each `PaneContainer`. `dropEntered` classifies the payload via
  `DropPayloadClassifier`; if `nil` (unsupported) → `false`, no overlay (validate-then-drop).
  `dropUpdated(info:)` maps `info.location` through `PaneDropZoneLayout.zone(at:)` → `@State activeZone`;
  `PaneDropOverlay` draws the 5 blobs and saturates the active one (warm-grey → olive-green). Disabled
  zones (green-half for file/URL) render muted + non-targetable (`DropActionResolver.resolve == nil`).
- **Verify:** unit — `PaneDropZoneLayoutTests` (zone geometry + `zone(at:)`), `DropPayloadClassifierTests`
  (type routing). GUI — Phase-3 HW: `scripts/check-macos.sh` + cua drag a Finder file over a pane,
  screenshot the overlay + active-zone tint vs `drop-overlay-frame-action.png` (flag: the live overlay
  render is headless-unprovable → Phase-3 HW-fidelity target, do not fake a pixel proof).

### ES-E18-2 — Insert-Path injects the path; folder on New-Tab cd's (host-resolved note) — both
- `performDrop` → `DropActionResolver.resolve(zone:content:)` → actuate:
  - `injectText(path)` → `terminalModel.sendInput(Data(path.utf8))` VERBATIM (focused pane).
  - `newTabCd(folder)` → `store.openWebPane`? no — `store.openChooserPane(.newTab)` is the ⌘T path; for a
    DIRECT terminal new-tab rooted at the folder use `store.newTab(kind:.terminal)` then send
    `LinkActionPolicy.changeDirectoryCommandLine(folder)` (a dropped FILE cd's to its PARENT via that
    helper — never bare `cd '<file>'`). THEN `overlayCoordinator.pushToast(Toast(id:"drop-cwd", flavor:
    .attention, title:"cd'd on host", body:"<path> is resolved on the host; it may not exist there."))`
    — the host-resolved-path note (the pane itself is local; the session it operates on is remote →
    advisory, never block).
  - `hostOpen(path)` (Open-In-Place) → `await metadataClient.openPath(path)`; on `false` push an error
    toast (never a silent no-op, the E10 idiom).
- **Verify:** unit — `DropActionResolverTests` (zone×content table). The cd idiom is already pinned by
  `LinkActionPolicyTests`; add a `WorkspaceStore`-level test that `newTabCd` results in a `cd … || cd
  <parent>` send (inject a fake terminal sink). GUI — Phase-3 HW: drop a folder on New-Tab, confirm the
  shell `cd`s + toast.

### ES-E18-3 — drag a sidebar tab to reorder with an insertion-line indicator — both
- The reorder itself is DONE (E6). WI-7 adds the insertion-line via `.dropDestination(…, isTargeted:)`:
  a 2pt accent rule on the targeted row's top edge. By-uuid drag, search/grouping gating, foreign-drop
  rejection are UNCHANGED.
- **Verify:** unit — `TabDragPayloadTests` already pins `resolveMove` (by-uuid, mid-drag `.updated`
  reshuffle, OOB/self no-op) — extend with a case asserting the targeted-row resolution. The insertion
  line is a pure overlay → assert the `isTargeted` state flips a published flag (ViewInspector-free:
  test the small `DropInsertionIndicator` model if extracted). GUI — Phase-3 HW: drag a rail tab,
  screenshot the insertion line.

### ES-E18-4 — open a URL in a built-in web pane (WKWebView, non-persistent store, no autoplay) — GUI-verifiable
- URL drop / `splitWeb` / `openWeb` → `store.openWebPane(url:placement:)` creates a `.web` leaf;
  `WebLeafView` renders the address-bar chrome + `WebRendererFactory.make`. The app target registers the
  factory at launch (like `VideoWindowFactory.shared`): the production view builds a `WKWebView` with
  `configuration.websiteDataStore = .nonPersistent()` (D9, no on-disk cookies/cache),
  `mediaTypesRequiringUserActionForPlayback = .all` (no autoplay), `javaScriptEnabled` default. The
  headless library renders the placeholder. `WebURLNormalizer` maps a bare host → `https://`, free text →
  DuckDuckGo, and validate-then-drops a malformed/non-http(s) string.
- **HANG-SAFETY (binding):** WKWebView is GUI/WebKit → never instantiated in a test or headless build; it
  lives ONLY in the app target behind `WebRendererFactory`. Tests exercise only the pure parts
  (`WebURLNormalizer`, `PaneKind.web` plumbing/persistence, the drop geometry).
- **Verify:** unit — `WebURLNormalizerTests`, `WebPaneStoreTests`, `PaneKindWebDecodeTests`. GUI —
  Phase-3 HW: register the factory in the app target, drop a URL, screenshot the rendered page vs
  `web-broswer.png` (live WKWebView render is headless-unprovable → Phase-3 target).

## Binding scope exclusions (from E18-carryovers — hard NO)
