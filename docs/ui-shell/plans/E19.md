# E19 — Window options (pin · window-size modes · multi-session UI)

> Implementation plan. **Verify-not-rebuild, fully client-side, NO wire change.** Honors
> `docs/ui-shell/plans/E19-carryovers.md` in full: ES-E19-4 (horizontal tab bar / `layout` selector /
> `auto-hide-tab-bar`) is **DROPPED** — out of scope, do NOT build it. E19 = **A29 (window-size) + A30

## Scope flags

- **touchesWire: NO.** Entirely client-side: `NSWindow.level`, client-window sizing, sidebar auto-hide,
  session switcher (`selectSession` already exists). `golden-check.sh` MUST show **33 emitted keys
  byte-identical + 13 frozen intact** (zero drift). Add NO wire message; no host round-trip.
- **touchesIOS: YES (shared ClientUI).** Pin / window-size are macOS-`NSWindow` concepts → `#if os(macOS)`
  gated, no-op on iOS (documented, not a dead toggle). The **session switcher** + the **auto-hide policy**
  are shared `SlopDeskClientUI`/`SlopDeskWorkspaceCore` and MUST render on iPad. The gate MUST run
  `bash scripts/check-ios.sh` — `swift build` on macOS will not type-check the `#if os(iOS)` slice.
- **touchesFloatCodec / controller: NO** — but the new window-size math is a **pure unit-tested function**
  written with the codec idioms anyway (ordered ternary min/max, separate `*` then `+`, never
  `addingProduct`/`fma`), mirroring `Sources/SlopDeskVideoHost/WindowPlacement.swift`
  (`WindowPlacementMath.placement`).

### WI-1 — Window-size DOMAIN: pure math + mode enum + settings keys (headless, SlopDeskWorkspaceCore)

**New files**

- `Sources/SlopDeskWorkspaceCore/Workspace/Domain/Tree/WindowSizeMode.swift`
  - `public enum WindowSizeMode: String, Codable, Sendable, CaseIterable { case remember; case grid; case
    frame }` — raw values are `remember`/`grid`/`frame`. Doc-comment the mapping like
    `NewTabPosition.swift`.
- `Sources/SlopDeskWorkspaceCore/Workspace/Domain/Tree/WindowSizeMath.swift`
  - `public enum WindowSizeMath` with PURE functions (no SwiftUI/AppKit import; `CoreGraphics` only):
    - `clampCols(_:) -> Int` / `clampRows(_:) -> Int` — `max(1, min(raw, 1000))` (validate-then-drop: never
      0; cap sane).
    - `clampPx(_:) -> Int` — `max(64, min(raw, 16384))`.
    - `gridContentSize(cols:rows:cell:) -> CGSize` — `width = cell.cellWidth * Double(clampCols(cols))`,
      `height = cell.cellHeight * Double(clampRows(rows))`. **Separate `*` then assignment, NEVER fma.**
      Returns raw (unclamped-to-screen) content px; the screen clamp is a separate step.
    - `clampToScreen(_ size: CGSize, visible: CGRect, chromeInsets: CGSize) -> CGSize` — clamp a desired
      CONTENT size so `content + chrome ≤ visible`, using ordered ternary min (`a < b ? a : b`, NaN-
      faithful, matching `WindowPlacementMath`). Floor at a sane min (e.g. 200×120). Pure.
    - `resolvedContentSize(mode:cols:rows:widthPx:heightPx:cell:visible:chromeInsets:) -> CGSize?` — the
      single entry the glue calls: `.remember → nil` (autosave path, no explicit size); `.grid →
      clampToScreen(gridContentSize(...), …)`; `.frame → clampToScreen(CGSize(clampPx(widthPx),
      clampPx(heightPx)), …)`. `nil` signals "let the autosaved frame stand".
  - All ordered comparisons; **no `addingProduct`/`fma`**; no force-unwrap.

**Edited files**

- `Sources/SlopDeskWorkspaceCore/Workspace/Store/SettingsKey.swift`

  - Add typed `Defaults.Keys`: `windowSize = Key<WindowSizeMode>(SettingsKey.windowSizeKey, default:

    Key<Int>(…, default: 24)`, `windowWidthPx = Key<Int>(…, default: 1000)`, `windowHeightPx =

  - Add `extension WindowSizeMode: Defaults.Serializable, Defaults.PreferRawRepresentable {}` (repairs a
