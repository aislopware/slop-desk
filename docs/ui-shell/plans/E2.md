# E2 — Overlay host mount (palette / cheat-sheet / find / toasts / connect / remote-picker)

Status: PLAN. Epic E2, depends on E1 (done). Pri 1, est L.

## Goal (from BACKLOG.md)

Mount the fully-built `OverlayCoordinator` into the live scene, pass its toggle closures into
`WorkspaceKeyDispatcher`, and build the three missing overlay views — **PaletteView**,
**KeyboardCheatSheetView**, **toast host** — plus wire the **connect-to-host** overlay and the
**remote-window picker** modal. This is a structural pass: the data layer (mixer, fuzzy matcher,
catalog, coordinator state machine, Toast model, RemoteWindowModel, AppConnection form) all already
exist and are tested. E2 only adds the SwiftUI view layer + the single mount/inject point. The Find
**bar** view itself lands in E5 — E2 only ensures `⌘F` is reachable (it already routes to
`store.requestFindInActivePane()` via the default `route` arm; no new view here).

## Non-obvious constraints

- **Single chord owner.** The app-level `WorkspaceKeyDispatcher` `NSEvent` `.keyDown` monitor owns ⌘⇧P
  and ⌘/ (and swallows them). The palette/cheat overlays must NOT add their own `.keyboardShortcut` for
  these — they receive the toggle via the coordinator closure threaded into the dispatcher.
- **In-overlay keys.** While the palette is visible its search field is focused, so typing reaches the
  field. ↑/↓/↩/⌘↩/Esc must be intercepted by the overlay (SwiftUI `.onKeyPress` phases on macOS 14+ /
  `.onExitCommand` / `.onMoveCommand`). The app NSEvent monitor passes bare arrows/Return through (it
  only swallows the prefix + bound chords), so they reach the focused overlay — good.
- **Headless-first.** Views compile in `SlopDeskClientUI` (no VT/SCStream/Metal). Tests must not
  instantiate any video/Metal object. The PaletteView/cheat/toast logic is testable via the coordinator
  (pure `@Observable`) + `ImageRenderer` opt-in snapshot (the existing `SlateSnapshotRender` pattern).
- **iOS rots silently.** New views live in shared `SlopDeskClientUI`; `#if os(macOS)` only around the
  `NSEvent`/AppKit bits. The overlay host must compile for iOS (the modal/sheet presentation differs).
  Run `bash scripts/check-ios.sh` after.
- **No wire/codec/float changes.** This epic is pure view + app-wiring. `touchesWire=false`,
  `touchesFloatCodec=false`, `touchesIOS=true` (shared UI compiled for iOS).

## Ordering rationale

WI-1 establishes the live coordinator + env + dispatcher toggles (everything else needs a mounted,
injected coordinator). WI-2/2a, WI-3, WI-4 build the three independent views (each only needs the
coordinator). WI-5 composes them into the host + adds the connect/picker views and the mount points
(needs all three views to exist). WI-6 pins behavior (write the coordinator-level tests first — they fail
against today's missing `rankedResults`/`acceptSelectedKeepingOpen`, proving the gap, per test-first
discipline). SlopDeskClientApp.swift, WorkspaceRootView.swift, and OverlayCoordinator.swift are each
edited by more than one WI — keep the WI order so the edits stack cleanly.
