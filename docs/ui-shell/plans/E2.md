## Goal (from BACKLOG.md)

Mount the built `OverlayCoordinator` into the live scene, pass its toggle closures into
`WorkspaceKeyDispatcher`, and build the three missing overlay views — **PaletteView**,
**KeyboardCheatSheetView**, **toast host** — plus wire the **connect-to-host** overlay and the

E2 only ensures `⌘F` is reachable (already routes to `store.requestFindInActivePane()` via the default
`route` arm; no new view).

## Non-obvious constraints

- **Single chord owner.** The app-level `WorkspaceKeyDispatcher` `NSEvent` `.keyDown` monitor owns ⌘⇧P
  and ⌘/ (and swallows them). The palette/cheat overlays must NOT add their own `.keyboardShortcut` for
  these — they receive the toggle via the coordinator closure threaded into the dispatcher.
- **In-overlay keys.** While the palette is visible its search field is focused, so typing reaches the
  field. ↑/↓/↩/⌘↩/Esc must be intercepted by the overlay (SwiftUI `.onKeyPress` on macOS 14+ /
  `.onExitCommand` / `.onMoveCommand`). The app NSEvent monitor swallows only the prefix + bound chords,
  so bare arrows/Return pass through to the focused overlay — good.
- **Headless-first.** Views compile in `SlopDeskClientUI` (no VT/SCStream/Metal). Tests must not
  instantiate any video/Metal object. Palette/cheat/toast logic is testable via the coordinator (pure
  `@Observable`) + `ImageRenderer` opt-in snapshot (the existing `SlateSnapshotRender` pattern).
- **iOS rots silently.** New views live in shared `SlopDeskClientUI`; `#if os(macOS)` only around the
  `NSEvent`/AppKit bits. The overlay host must compile for iOS (modal/sheet presentation differs). Run
  `bash scripts/check-ios.sh` after.
- **No wire/codec/float changes.** Pure view + app-wiring. `touchesWire=false`,
  `touchesFloatCodec=false`, `touchesIOS=true` (shared UI compiled for iOS).

injected coordinator). WI-2/2a, WI-3, WI-4 build the three independent views (each only needs the
coordinator). WI-5 composes them into the host + adds the connect/picker views and mount points (needs
all three views). WI-6 pins behavior — write the coordinator-level tests first (they fail against today's
missing `rankedResults`/`acceptSelectedKeepingOpen`, proving the gap, per test-first discipline).
`SlopDeskClientApp.swift`, `WorkspaceRootView.swift`, and `OverlayCoordinator.swift` are each edited by
more than one WI — keep the WI order so edits stack cleanly.
