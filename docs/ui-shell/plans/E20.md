# E20 — CLI parity + watch + first-launch — Implementation Plan

> The FINAL UI-shell epic. Two halves: (1) a full `slopdesk` user-facing CLI mapping
> the designed subcommand surface onto the existing control plane, and (2) a guided first-launch
> flow composing already-built settings. Acceptance ES-E20-1…4.
>
> **Audit verdict (carry-over §0/§6 first job — DONE):** the existing CLI spine is
> `SlopDeskCtlCore/CtlCore.swift` (pure line-protocol + param builders, 199 LOC) +
> `Sources/slopdesk-ctl/main.swift` (socket + dispatch, 686 LOC) + host
> `SlopDeskHost/AgentControlListener.swift` (AF_UNIX NDJSON server, verbs list-panes/read/
> write/run/wait/spawn/kill/resize/report/subscribe). **This host socket drives PTYs/agents on
> the HOST. It cannot drive the CLIENT GUI** (window/tab/pane/jump/view/config/theme live in
> `WorkspaceStore` inside the client app). The designed CLI talks to the running app over a
> runtime control socket; the faithful equivalent is a **new CLIENT-side control socket** owned
> by the running client app, reusing the SAME CtlCore NDJSON line protocol. The new `slopdesk`
> binary is a **superset front-end** that talks to the client socket for GUI ops and is
> self-contained for local ops (`watch`/`version`/`completions`/config-file). `slopdesk-ctl`
> and the host socket are UNCHANGED (still used by agent hooks / in-pane agents).

## Wire / iOS / float-codec flags

- **touchesWire = false.** The NDJSON control protocol (host AND new client socket) is *not* the
  golden-frozen binary wire. Golden freezes terminal/video/inspector binary paths only. Adding
  control methods does not touch `golden/golden_vectors.json`. **Expected golden diff = ZERO; 13
  frozen keys intact.** No `docs/20-wire-protocol.md` change.
- **touchesIOS = true.** First-launch composes shared `SlopDeskClientUI` + `SlopDeskWorkspaceCore`
  (On-Launch, theme, agent-install). The `/usr/local/bin` install + OS integration + completions
  are `#if os(macOS)`. **Run `bash scripts/check-ios.sh` after WI-9.**
- **touchesFloatCodec = false.** No codec/controller/float-math touched. Frecency scoring is
  integer/ordered (existing `FolderFrecency`, CLAUDE.md #2 already honored). The watch OSC builder
  is byte/string only.

## Architecture: one user-facing CLI, two transports

```
 slopdesk <subcommand>            (NEW exe; superset of slopdesk-ctl)
   ├─ local/pure ops ────────────── version, completions, watch, config get/set (file)
   │                                 → no socket, self-contained
   ├─ client GUI ops ────────────── windows/tabs/panes, window/tab/pane, tab badge,
   │   (NEW client socket)           pane capture/send-keys, jump/learn/ignore, view/edit,
   │                                 open <recipe>, config reload/show, theme/font/keybind list,
   │                                 watch:claude  → SlopDeskClientControl (running client app)
   └─ legacy/agent ops ──────────── ipc, state:<agent>  → host socket (SLOPDESK_CONTROL_SOCKET)
                                      (unchanged; delegate to existing slopdesk-ctl behavior)
```

Socket resolution for client ops: `--socket` > `SLOPDESK_CLIENT_SOCKET` env > default
`~/Library/Application Support/SlopDesk/cli-control.sock`. If unreachable for an op that requires
the app → exit non-zero with "requires a running SlopDesk app" (per `reference__cli.md`).

**Headless-first split (hang-safety):** all parse/dispatch/format/exit-code logic is PURE in
libraries and exhaustively unit-tested with fakes (revert-to-confirm-fail). The AF_UNIX accept loop
(`ClientControlServer`) and the watch subprocess spawn live in compiled-only sites and are NEVER
instantiated in a test (mirrors `AgentControlListener` exclusion). No SCStream/VT/Metal/NSWindow/
WKWebView in any test.

## Per-story approach & verification

### ES-E20-1 — `open/view/edit/config/font/theme/keybind/tab/pane/window` drive the app; `--json` structured
- **Approach.** WI-2/3/4. New `slopdesk` subcommands serialize a `ClientControlProtocol` request, send to the client socket, render the response as table or `--json`. `window/tab/pane` + plural `windows/tabs/panes` (= `… list`) read `WorkspaceStore` via `WorkspaceControlBackend`. `config get/set/unset/edit/show/path/validate/reload` map onto `EnvConfig`+`PreferencesStore`: `--transient` writes `EnvConfig.overlay` without persisting; `reload` broadcasts the config-change notification. `font/theme/keybind list` enumerate `ThemeStore`/libghostty font list/`WorkspaceBindingRegistry`. `view/edit` → WI-6 shim. `--json`/`--format json` switches every list/inspect path to JSON; `--no-headers` strips table headers.
- **Verify.** Unit: `CLIArgsTests`, `CLIFormattingTests` (table + `--json` golden strings, `--no-headers`), `ClientControlDispatcherTests` (each method, fake backend). GUI (Phase-3): run a live client + `slopdesk panes --json`, `slopdesk config set font-size 14 --transient`, `slopdesk theme list`.

### ES-E20-2 — `watch <cmd>` spinner→badge; `watch:claude` exit 0/4/9
- **Approach.** WI-7: `slopdesk watch <cmd>` runs `<cmd>` as a subprocess in the current pane; emits OSC 9;4 indeterminate (spinner) at start, then `clear` (success) or `error` (exit≠0) on completion (`WatchProgress` byte builders → printed to the controlling terminal, rendered by the pane's existing `ProgressState` path), then posts the "Notify on Watch Finish" system notification (existing OSC-9 notification path) unless `-q`/`--quiet`. WI-8: `slopdesk watch:claude <id>` polls the client `agent-status` method; `WatchClaudeOutcome` maps idle/closed→0, id-never-seen→4, timeout→9.
- **Verify.** Unit: `WatchProgressTests` (exact OSC bytes per state; exit→state mapping), `WatchClaudeOutcomeTests` (all three exit codes, revert-to-confirm-fail). GUI (Phase-3): real `watch sleep` spinner→badge in a live terminal; `watch:claude` against a live claude pane.

### ES-E20-3 — `tab badge --kind`, `pane capture`, `jump/learn/ignore`, `version`, `completions`
- **Approach.** `tab badge --kind <running|completed|finished|unread|error|awaiting-input>` → client `tab-badge` method → `TabBadgeKind` (map `unread`→a documented closest case; rest 1:1). `pane capture --lines N` → client `pane-capture` reads the pane's client-side scrollback/replay buffer, last N lines. `jump/learn/ignore` (WI-5) reuse `FolderFrecency`/`FolderFrecencyStore`: `jump [query]` resolves a path via `JumpResolver` then sends `cd <resolved>\n` VERBATIM via `pane-send-keys` (no query toggles `$HOME`↔last-jump; `--no-cd` prints only); `learn [path]` records a visit (no-arg uses the client's cached OSC-7 pane cwd); `ignore <path>` calls `forget`. `version` = `CLIVersion.versionSummary()` (no socket). `completions <shell>` = `CLICompletions` (no socket).
- **Verify.** Unit: `JumpResolverTests` (rank + toggle + `--no-cd`), `CLICompletionsTests` (per-shell golden; Claude-only), `CLIVersionTests`, dispatcher tests for `tab-badge`/`pane-capture`. GUI (Phase-3): `slopdesk jump <dir>` cd's the focused pane; `slopdesk tab badge --kind running`.

### ES-E20-4 — first-run: On-Launch, install CLI, install Claude hooks
- **Approach.** WI-9: `FirstLaunchModel` gates first-run presentation (`hasCompletedFirstLaunch` via `Defaults`). `FirstLaunchView` composes existing settings per the 6 screenshots: step 1 On-Launch picker (`@Default(.onLaunch)`), step 2 Default-Terminal (LOCAL handler via `DefaultTerminalIntegration`; "for Common Apps"/remote honestly-disabled with a documented note — NO dead button), step 3 Install-CLI (`CLIInstaller` symlink + Omit-prefix + Allow-Overwrite), step 4 Theme (grid + palette entry), step 5 Install-Claude-hooks (reuse `AgentHooksController`, Claude only). iOS keeps cross-platform steps (On-Launch, theme, agent-install); macOS-only steps `#if os(macOS)`.
- **Verify.** Unit: `FirstLaunchModelTests` (gating/step completion). GUI (Phase-3, HW): the 6 screenshots — first-launch window, On-Launch dropdown, OS-integration buttons, theme picker palette + grid, agents behavior list; real `/usr/local/bin/slopdesk` install + admin prompt. `bash scripts/check-ios.sh` BUILD SUCCEEDED.

## Exclusions honored (carry-over §4)
- No `watch:codex`/`watch:opencode`, no Codex/OpenCode hooks; `watch:<agent>` ships **`watch:claude` only**; completions/help never list non-Claude agents.
- "Set as Default Terminal for Common Apps" rewriting REMOTE editor configs → **local editors only**; remote case honestly-disabled with a documented note (no dead button).
- `view`/`edit` = a **shim** that sends `less <path>` / `$EDITOR <path>` / `open <url>` VERBATIM to a new split pane — NOT a native local renderer (documented).
- `import`/`export` of ghostty/kitty/alacritty **non-theme config** and `features` demos → **deferred** with a documented note (theme import already shipped E15); do not block ES-E20-1…4.
- SSH-host / nested-SSH badge → **out** (every pane is already remote).
- send-keys / jump / cd / recipe-command literal text = **VERBATIM UTF-8**, named keys via keycode path, **never SendKeysParser**. No app-layer crypto. `.slopdeskrecipe`/config/socket input validate-then-drop, no force-unwrap, bound counts before allocating.
