# E3 — Tree-path domain completion — Implementation Plan

> Epic E3 (BACKLOG.md `## E3 —`). Goal: fix canvas-only dead seams and missing tab/cwd/close policies on the
> **live tree path** (`WorkspaceStore.liveModel == .tree`, slopdesk's UI shell).
> specRef: `spec/user-interface__window-tab-split.md`. Acceptance: ES-E3-1 … ES-E3-5. dependsOn E1 (done).
>
> Verified file-by-file against code on disk. **Pure domain work**: new policy value-types + a tree-path
> reopen LIFO + a pure pane-cycle op, all unit-testable headless, plus thin store wiring. **No wire bytes,
> no codec/float, no SCStream/VT/Metal.**

## TL;DR — what's real on disk (the gaps E3 closes)

- **A4 reopen-last-closed is canvas-only & DEAD on tree.** `WorkspaceStore+PaneCycle.swift:reopenLastClosedPane()`
  is an intentional empty stub (E1 left it for E3). Canvas `recentlyClosed` single-slot
  (`WorkspaceStore.swift:563`, `reopenClosedPane()` `:635`) restores via `workspace.canvas.restoring(...)`
  — never touched by `closePaneTree`/`closeTab`. Routing is live: `WorkspaceBindingRouting.swift:167`
  `.reopenClosed → store.reopenLastClosedPane()`. **Gap: a tree-path LIFO of closed TABS + restore.**
- **A26 cwd-inheritance is missing.** `splitActivePane` (`:2393`), `newTab(kind:)` (`:2451`), and
  `openChooserPane` (`:3840`) all build `PaneSpec(kind:title:)` with `lastKnownCwd: nil`, never reading the
  active pane's cwd. E4 added `MetadataClient.cwd()` (pull) + `setLastKnownCwd(_:for:)` (`:3866`), but only
  the Details-Panel Info tab calls it (`InspectorColumn.swift`), so `lastKnownCwd` is only fresh while the
  inspector is open. **Gap: a `working-directory` policy (inherit/home/path) + a fresh cwd source + a
  deferred `cd` into the new pane.**
- **A17 new-tab-position is missing.** `WorkspaceTreeOps.newTab` (`:586`) and `breakPaneToTab` (`:709`)
  unconditionally `tabs.append(...)`. **Gap: a `new-tab-position` (auto/end/after-current) policy.**
- **A27 close-confirm policy is missing.** Every close guard (`requestClosePane` `:575`,
  `requestClosePaneTree` `:589`, `requestCloseActivePaneTree` `:2781`, `confirmPendingClose` `:603`) checks
  ONLY `registry[id]?.isShellBusy == true`. No `always`/`multiple_tabs` policy, no window-close confirm.
  **Gap: a `CloseConfirmationPolicy` (process/always/multiple_tabs) feeding the guard, split for tab vs window.**
- **A10 sequential pane-cycle has a STORE method but NOT a pure op.** E1 added
  `WorkspaceStore.cyclePaneFocusTree(forward:)` + internal `paneCycleTreeTarget(forward:)`
  (`WorkspaceStore+PaneCycle.swift`, pinned by `CyclePaneFocusTreeTests`). `WorkspaceTreeOps` has only
  `moveFocus(.next/.previous)` (via `FocusResolver.cycle`). **ES-E3-5 wants a PURE op in `WorkspaceTreeOps`
  covered by tests** — extract one, have the store delegate to it.

## Flags (CLAUDE.md compliance)

- **Wire protocol / golden corpus — NOT touched.** `touchesWire = false`. No wire message, no codec change.
  The cwd-inherit `cd <path>\n` rides the EXISTING PTY `sendBytes` path (same `SessionTemplateEngine.launchBytes`
  route templates use). The cwd-freshness refresh rides the EXISTING E4 metadata channel
  (`MetadataClient.cwd()`), no new verb. **`make golden` must be a no-op diff — run it to PROVE no byte moved.**
  `docs/20-wire-protocol.md`: **no change needed** (state so).
- **Float-math / codec / controller — NOT touched.** `touchesFloatCodec = false`. No FMA, no float compares,
  no FEC. (Existing `Double.minimum`/`Double.maximum` index-clamp idiom in `WorkspaceTreeOps` preserved
  verbatim where `insertTab` reuses the tab-index clamp pattern.)
- **iOS (`#if os(iOS)`) — shared modules compile on iOS.** `touchesIOS = true`. Every new value-type
  (`NewTabPosition`, `WorkingDirectoryPolicy`, `CloseConfirmationPolicy`, `RecentlyClosedTab`) and every
  `WorkspaceTreeOps`/`WorkspaceStore`/`SettingsKey` edit lives in `SlopDeskWorkspaceCore`, which **compiles
  into the iOS slice**. macOS `swift build` won't catch an iOS type error → **`bash scripts/check-ios.sh` is
  mandatory.** ONLY macOS-only surface: the window-close NSWindow interception (WI-4, app layer), `#if
  os(macOS)`, adds no iOS code.
- **Validate-then-drop / no force-unwrap.** All new policy parsers (`WorkingDirectoryPolicy(rawConfig:)`, the
  close/new-tab enum `init?(rawValue:)`) return a sane default for an unknown stored string — no `!`, no trap.
  The reopen-LIFO restore validates the active session still exists before inserting (no force unwrap of
  `activeSessionIndex`). `RecentlyClosedTab` is **in-memory only** (not persisted, matching canvas
  `recentlyClosed`) → no untrusted-decode surface.
- **No app-layer crypto / no version negotiation.** Unaffected.

## Traps specific to E3

- **The settings UI rows are E7, not E3.** E3 adds the SettingsKeys + accessors (consumed now by the domain);
  the Shell/General settings rows that bind them are E7 scope. Don't build settings panes here.
- **`recentlyClosedTabs` is IN-MEMORY only** (matching canvas `recentlyClosed` `:559`) — do NOT persist it (no
  untrusted-decode surface; a relaunch already restores the live layout).
- **Capture the tab BEFORE the close op mutates the tree** — `closeTab`/`closePane` cascade can drop the whole
  session; read `Tab` + its specs from the pre-mutation `tree`.
- **`.home` resolves to `nil` cwd** (no `cd` sent → host shell's login cwd). A literal `cd $HOME` would fight a
  shell that already starts at `$HOME`; document the nuance rather than emit a redundant `cd`.
- **Keep `newTab(at: .end)`/`.auto` byte-identical to `tabs.append`** so every existing `newTab`/`breakPaneToTab`/
  `addSystemDialogPane` call site is unchanged (the system-dialog ephemeral tab must still land predictably).
  Default the param to `.end`.
- **Don't double-source cwd.** Inherit value comes from the active pane's `lastKnownCwd`; the freshness refresh
  writes the SAME field via the EXISTING `setLastKnownCwd` dirty-guard (`:3872`) — no new field, no reconcile
  churn when unchanged.
- **iOS rot:** every new type compiles into the iOS slice but macOS `swift build` won't type-check it —
  `scripts/check-ios.sh` is mandatory after WI-1…WI-5.
