# E4 — Host metadata RPC service — Implementation Plan

> Epic E4 (BACKLOG.md `## E4 —`). Goal: ONE shared host control-channel RPC listing per-pane
> **processes**, **listening ports**, **cwd**, **git status/branch/diff**, **directory contents
> (lazy)**, and **Claude/agent session files** — the backend for every remote-dependent Details-Panel
> surface. specRefs: `spec/user-interface__details-panel.md`, `spec/agents__history.md`,
> `spec/terminal-features__shell-integration.md`. Acceptance: ES-E4-1 … ES-E4-5. dependsOn: —.
>
> Written against the actual code on disk. E4 is greenfield (none of the six verbs exist) but slots
> into an EXISTING request/response precedent (`requestBlockOutput` 15 → `blockOutput` 29) and reuses
> finished engines.

## TL;DR — the shape of the work

The Details Panel reads host-side metadata living on the **remote** machine; slopdesk must proxy it
over the wire. Design: **one generic request/response pair on the CONTROL channel** (exact
`requestBlockOutput`/`blockOutput` precedent) carrying a **verb byte + per-request correlation id +
length-prefixed payload**. Each verb's payload is a separate **manual-binary** sub-codec
(`MetadataCodec`) — validate-then-drop, count-before-alloc, no `!`. The host runs OS queries behind a
thin reviewed `HostMetadataProbe` shim (git/lsof/proc/FileManager), split from a **pure**
`MetadataResponseBuilder` unit-tested with injected fakes (no subprocess, headless). The client gets a
`requestMetadata(...)` round-trip with a pending-request registry that **never hangs** (5 s timeout →
empty/error, like `TerminalBlockModel`), typed decoders, per-pane caches + `@Observable` models, and
minimal rendering in the EXISTING `InspectorColumn` Info/Git/Files tabs (polished 4-tab chrome + ⌘⇧R
is a later visual epic; E4 fills the data).

**Why a generic envelope, not ~8 verb-specific wire types:** eight frozen control types = eight golden
keys + eight codecs on the wire forever. Two types (`metadataRequest`/`metadataResponse`) keep the
wire surface tiny; per-verb payloads ride opaquely inside (like `blockOutput`'s raw bytes), pinned by
their OWN codec golden samples + unit tests. Honors "manual binary, never JSON/Codable" for the whole
path including inner payloads, and keeps `make golden` churn to two envelope keys plus representative
inner samples.

## Flags (CLAUDE.md compliance)

- **Wire protocol / golden corpus — TOUCHED. `touchesWire = true`.** Two new CONTROL types:
  client→host **type 16 `metadataRequest`**, host→client **type 30 `metadataResponse`** (next free:
  client→host 10–15 used → 16; host→client 20–29 used → 30). **Intended additive change within
  protocol version 1** (unknown type DROPS the frame via `unknownMessageType`, never traps, like
  26/27/28/29). Must: edit `WireMessage` + encode/decode; add golden vectors via `slopdesk-corevectors`
  (NEW `metadataWireMessages` group) + representative `MetadataCodec` payload samples; **hand-merge**
  the regenerated subset into `golden/golden_vectors.json` so the 13 frozen XCTest-pinned keys survive
  (NEVER redirect the generator over the corpus); update `docs/20-wire-protocol.md` §4 message table +
  field notes (new "next free": client→host 17, host→client 31). Generate with **NO `SLOPDESK_*` env
  set**. All multi-byte ints big-endian; strings UTF-8; no JSON/Codable on the wire.
- **Float-math / codec / controller — NOT touched. `touchesFloatCodec = false`.** No FEC, FMA, or
  float compares (the metadata codecs are integer/byte length-prefix codecs).
- **iOS (`#if os(iOS)`) — TOUCHED. `touchesIOS = true`.** `WireMessage`, `MetadataCodec`,
  `MetadataClient`, typed models, and `SlopDeskClient.requestMetadata` all compile into the iOS slice
  (shared modules). iOS render surface differs (sidebar → sheet/drawer per details-panel mapping); E4
  only needs the data to reach the iOS models, but the shared decode path rots silently on a macOS
  build. **Run `bash scripts/check-ios.sh` after every WI touching a shared module.** The host probe
  (`HostMetadataProbe`) is `#if os(macOS)` (spawns `git`/`lsof` + Darwin `proc_*`), never compiled for
  iOS.
- **Validate-then-drop on untrusted input.** Every decoder (envelope AND each `MetadataCodec` payload)
  validates declared `requestID`/`verb`/`status`/lengths and every nested count **before allocating**,
  reads length-prefixed fields via `BigEndianReader.readBytes` (throws `truncated`, never over-reads),
  requires strict UTF-8 (`malformedBody` else), reads interop booleans as `byte != 0`, and **never
  force-unwraps**. Unknown verb/status is carried/clamped forward-tolerantly (host replies
  `status = unsupportedVerb`; client treats unknown status as error) — malformed datagram dropped,
  round-trip times out to empty, never a trap. **The host also treats the client's request `payload`
  (paths, ids) as untrusted**: path args confined to the pane's cwd subtree (reject `..` escapes /
  absolute paths outside repo root), counts/sizes capped (dir listing ≤ 4096 entries, diff/session
  bytes ≤ the 16 MiB frame cap), so a hostile `listDirectory("/etc")` /
  `readAgentSession("../../secrets")` can't exfiltrate arbitrary host files.
- **No app-layer crypto / no version negotiation.** Unaffected — rides the SAME trusted-mesh CONTROL
  channel; no tokens/pairing. Host still accepts only protocolVersion 1.

## Architecture decision: one generic RPC pair on the CONTROL channel

```
WireMessage.metadataRequest(requestID: UInt32, verb: UInt8, payload: Data)   // type 16, client→host, CONTROL
  body = [UInt32 BE requestID][UInt8 verb][UInt32 BE payloadLen][payload bytes]

WireMessage.metadataResponse(requestID: UInt32, status: UInt8, payload: Data) // type 30, host→client, CONTROL
  body = [UInt32 BE requestID][UInt8 status][UInt32 BE payloadLen][payload bytes]
```

- **`requestID`** — client-chosen monotonic UInt32 correlating a reply to one of several in-flight
  requests (Details Panel may fire processes + ports + gitStatus at once). Host echoes it verbatim
  (stateless responder, like `pong`).
- **`verb`** — selects the operation (`MetadataVerb` enum). Unknown → `status = unsupportedVerb`, empty
  payload.
- **`status`** — `0 ok / 1 notFound / 2 error / 3 unsupportedVerb`. Raw byte; unknown future status
  clamps to error client-side.
- **`payload`** — length-prefixed, validated before alloc; verb-specific `MetadataCodec` shape, OR raw
  opaque bytes for `gitDiff`/`readAgentSession`/`cwd`.
- **Pane identity** rides the **mux channel envelope** (channelID = pane/PTY), as for types 26/27 — so
  `processes`/`ports`/`cwd`/`gitStatus`/`gitBranch` need **no pane field** in the payload; only
  path/id-parameterized verbs carry an argument.

### Verbs (`MetadataVerb: UInt8`)

| verb | name | request payload | response payload (status ok) |
|---|---|---|---|
| 1 | `processes` | empty (pane) | `ProcessList` |
| 2 | `ports` | empty (pane) | `PortList` |
| 3 | `cwd` | empty (pane) | UTF-8 path string (opaque) |
| 4 | `gitStatus` | empty (pane cwd) | `GitStatus` (branch + remote + ahead/behind + changed files) |
| 5 | `gitDiff` | UTF-8 file path (repo-relative) | raw `git diff` bytes (opaque) |
| 6 | `listDirectory` | UTF-8 path (empty = pane cwd) | `DirListing` (one level, lazy) |
| 7 | `listAgentSessions` | UTF-8 project path (empty = pane cwd) | `AgentSessionList` |
| 8 | `readAgentSession` | UTF-8 session id/path | raw JSONL/JSON bytes (opaque) |

> **`gitBranch` is subsumed into `gitStatus`.** The Git tab header (branch, remote, +ahead/-behind) and
> the changed-file list render together; one `gitStatus` round-trip is faithful to the panel.

### `MetadataCodec` payload layouts (manual binary, all BE, validate-then-drop)

- **`ProcessList`** = `[UInt16 count]` then per entry `[UInt32 pid][UInt32 uptimeSec][UInt16 nameLen][name UTF-8]`.
- **`PortList`**   = `[UInt16 count]` then per entry `[UInt16 port][UInt8 proto (0 tcp/1 udp)][UInt16 nameLen][procName UTF-8]`. count 0 → "No listening ports".
- **`DirListing`** = `[UInt16 count]` then per entry `[UInt8 isDir][UInt16 nameLen][leafName UTF-8]`. Leaf names only; client joins with the request path (lazy per-expand).
- **`GitStatus`**  = `[UInt8 hasRepo]`; if repo: `[UInt16 branchLen][branch][UInt16 remoteLen][remoteURL][Int32 BE ahead][Int32 BE behind][UInt16 count]` then per file `[UInt8 statusCode][UInt16 pathLen][path UTF-8]` (statusCode = porcelain XY packed into one byte).
- **`AgentSessionList`** = `[UInt16 count]` then per entry `[UInt8 agentKind (0 claude/1 codex/2 opencode)][UInt16 idLen][id][UInt16 titleLen][title][UInt16 cwdLen][cwd][Int64 BE mtimeMS]`.
- `cwd` / `gitDiff` / `readAgentSession` carry no nested codec — raw UTF-8 / raw bytes (the envelope
  already length-prefixes the payload). Client parses `readAgentSession` JSONL via the EXISTING
  `SlopDeskInspector.TranscriptParser`.

Every count is validated against the reader's remaining bytes before the per-entry loop; every
length-prefixed field uses `readBytes` (throws `truncated`); no `!`; UInt16 length fields are clamped
on encode (reuse `clamped*` in `WireMessage+Encode`) so a >64 KiB field can't wrap.

## Approach per acceptance story + how to verify

- **ES-E4-1 (process list name/PID/uptime + listening ports → render in inspector) — both.** Verbs
  `processes`/`ports` → `ProcessList`/`PortList` codecs → `PaneMetadataModel` → `ProcessPortsView` in
  the Info tab. **Unit:** `MetadataCodecTests` (round-trip + truncation drop), `MetadataClientTests`
  (decode + cache), `MetadataResponseBuilderTests` (fake query → correct payload, "No listening ports"
  when empty). **GUI:** `scripts/check-macos.sh` — open the inspector on a live pane, confirm
  `-zsh`/agent rows with green dot + pid + uptime and the ports section.
- **ES-E4-2 (git branch, ahead/behind, changed-file list, file diff) — both.** `gitStatus` (folds
  branch+remote+ahead/behind+files) + `gitDiff(file)` (opaque bytes). **Unit:** `MetadataCodecTests`
  `GitStatus` round-trip incl. `hasRepo=false`; builder with a fake git query (porcelain → packed
  statusCode; rev-list → ahead/behind). **GUI:** Git tab shows `main` + remote + `+N −M` + file list;
  selecting a file overlays the unified diff (added green / removed pink) via `gitDiff`.
- **ES-E4-3 (lazy host directory listing on expand) — both.** `listDirectory(path)` one level; client
  joins leaf names with the request path; expanding a node fires another request. **Unit:** `DirListing`
  round-trip + count-before-alloc; builder confines path to the pane-cwd subtree (reject `..`/absolute).
  **GUI:** Files tab tree — collapsed dirs, click a disclosure triangle → children load; "Find" filters
  the fetched subtree client-side.
- **ES-E4-4 (list + read host Claude session JSONL for current project) — both.**
  `listAgentSessions(project)` enumerates `~/.claude/projects/<slug>/*.jsonl` (+ codex/opencode) →
  `AgentSessionList`; `readAgentSession(id)` streams raw JSONL → client renders via the EXISTING
  `TranscriptParser` in `AgentSessionHistoryView`. **Unit:** `AgentSessionList` codec round-trip; builder
  with a fake fs (path-slug mapping, mtime sort); a parse test feeding canned JSONL through
  `TranscriptParser` (assert bytes reach the parser unmangled). **GUI:** Info-tab "View Session History"
  opens a rendered transcript pane.
- **ES-E4-5 (every decoder validates declared counts/lengths before allocating, drops malformed without
  trapping) — unit-testable.** Cross-cutting hardening. **Unit:** for the envelope AND every
  `MetadataCodec` payload: declared count/len > remaining → throws `truncated` with NO allocation; bad
  UTF-8 → `malformedBody`; unknown verb/status byte tolerated (no trap); a fuzz-ish table of
  truncated/oversize/garbage buffers each returns nil/throws, never crashes. Mirror existing
  `claudeStatus`/`blockOutput` decode-guard tests. **Revert-to-confirm-fail:** each new guard test must
  FAIL if the length check is removed.

## Mapping notes — design choices that diverge from a local-terminal-only design (closest faithful equivalent)

- **cwd source.** A local-only terminal tracks cwd via OSC 7; slopdesk resolves it host-side via the
  foreground pid's vnode path (`proc_pidinfo` PROC_PIDVNODEPATHINFO) — no sniffer/wire change. OSC-7
  tracking in `HostOutputSniffer` is a clean future enhancement (feeds the SAME `cwd` verb /
  `lastKnownCwd`). Displayed path is always the HOST filesystem path — label it remote.
- **"Reveal in Finder" / "Open in VS Code/Cursor/Xcode/Typora".** Target a REMOTE host path; no local
  Finder/app can open it. **Deferred (P2)** — needs a remote-open/SSHFS story. E4 ships "Copy Path" only
  (pasteboard, client-side) + the in-app `RemoteFileTreeView` as the browse-the-host equivalent.
- **Git "Commit" / "Fork" buttons.** Mutating the host repo needs host-side command execution beyond
  read-only status. **E4 is read-only** (status + diff). Commit/Fork deferred (would send a PTY command
  or a new mutating verb — out of scope).
- **Agent "Resume" / "Send to Chat" / "Fork in…".** Listing + transcript rendering are E4; spawn-with-
  `--resume` and composer routing are the agent epics (history/composer). E4 provides the data + viewer,
  not Resume/Send-to-Chat.
- **iOS Details Panel.** A right sidebar can't sit beside the terminal on a phone — becomes a
  sheet/drawer (the visual epic). E4 only guarantees the metadata + models reach the iOS slice (compile
  via `check-ios.sh`); iOS panel chrome deferred.
- **The polished 4-tab Details-Panel chrome (⌘⇧R toggle, segmented header).** Half-present in
  `InspectorColumn` (Info/Git/Files tabs, empty states). E4 fills the DATA into those tabs; the
  pixel-faithful header/toggle is a separate visual epic.

## Traps specific to E4

- **Golden is NOT a no-op this epic.** Don't expect a clean `make golden` diff — hand-merge the two new
  keys surgically; NEVER redirect `slopdesk-corevectors` over `golden_vectors.json` (drops the 13 frozen
  keys). Regenerate with NO `SLOPDESK_*` env set.
- **Both control loops.** `MuxChannelSession` has TWO control inbound loops (initial ≈`:492`, reattach
  ≈`:751`). The `.metadataRequest` case must be in BOTH or metadata silently dies after a reconnect —
  same shape as the duplicated `requestBlockOutput` case.
- **Never hang the client.** `serveMetadata` must ALWAYS enqueue a response (status error/empty on any
  failure), and `MetadataRequestRegistry.await` must time out (5 s → empty) — the `TerminalBlockModel`
  rule. A pane switch mid-flight must cancel/ignore the stale requestID (correlate by id, not by pane).
- **Untrusted request payloads.** Treat the client's path/id args as hostile: confine to the pane-cwd
  subtree, reject `..`/absolute escapes, cap entry counts + byte sizes BEFORE reading. Tested in
  `MetadataResponseBuilderTests` (revert-to-confirm-fail on each guard).
- **iOS rot.** Everything except `HostMetadataProbe` compiles into iOS; a macOS-only `swift build` won't
  catch an iOS type error in the shared codec/models — `check-ios.sh` mandatory after WI-1/2/4.
- **Hang-safety.** Never spin `HostMetadataProbe`'s subprocess/PTY work in xctest — the pure builder +
  fakes are the test seam (PTYForegroundProbe precedent).
- **lsof parsing is hostile-input too** — the host PARSES `lsof` output; treat it defensively (malformed
  line → skip, never trap). Same for porcelain/rev-list output.
