### Design divergences (state explicitly)

1. **Global Search is a non-modal overlay surface, NOT a first-class results *tab/pane*.** A dedicated
   results tab (with the sidebar "TABS" header swapping to "SEARCH") was considered, but a real
   `PaneKind.searchResults` would blast-radius every `switch PaneKind` site (PaneContainer, reconcile,
   isVideo, persistence decode, chooser, sidebar rows, titlebar) and risks silent rot for an "M"
   epic. The chosen design: a **large, non-scrimmed `GlobalSearchView`** hosted by
   `OverlayCoordinator`, holding the query field + `Aa`/`.*` toggles + grouped results + summary,
   dismissable with Esc and **re-openable with ⇧⌘F** (which restores the last results in-memory).
   `PaneKind.searchResults` + the "SEARCH" sidebar header are noted as **future work**.
2. **In-buffer highlight + scroll-to-match come from libghostty** (`search:`/`navigate_search:`/
   `scroll_to_row`), while the `N of M` counter comes from `TerminalSearchController` over
   `scrollbackTextLines()`. These can **drift**: libghostty searches its full grid; our mirror is the
   `scrollbackTextLines()` snapshot taken when the bar opened. Documented behaviour: the counter is
   the mirror's count; the highlight is libghostty's. They agree in the common case (same buffer). The
   bar refreshes the mirror on open and on `Aa`/`.*` toggle.
3. **Regex semantics = ICU (`NSRegularExpression`), not Rust `regex`** — already the case in
   `TerminalSearchController`. `.*` labels regex mode; ICU supports lookaheads, Rust doesn't — minor,
   acceptable, document in the bar's help/tooltip.
4. **Global Search covers terminal scrollback only.** slopdesk has no file/folder pane, so there is
   no file-name / file-content index to search → terminal panes only. Sources from non-terminal
   (video) panes contribute no lines. Noted in spec mapping; future work.
5. **Empty/disconnected scrollback:** a pane that never received bytes contributes `[]` → it's simply
   absent from results. Acceptable per spec mapping notes.

### WI-1 — Pure `GlobalSearchController` engine + tests (WorkspaceCore, headless)

**New file** `Sources/SlopDeskWorkspaceCore/Terminal/GlobalSearchController.swift`:
- `public struct GlobalSearchSource: Sendable { let paneID: PaneID; let sessionID: …; let tabID: …;
  let groupTitle: String; let lines: [String] }` — one per live terminal pane.
- `public struct GlobalSearchHit: Equatable, Sendable { let source identity (paneID/sessionID/tabID);
  let line: Int; let column: Int; let length: Int; let excerpt: String; let highlight: Range<Int> }`
  — `excerpt` = the full matched line; `highlight` = UTF-16 column range within the excerpt for the
  amber highlight (mirrors `find.png`).
- `public struct GlobalSearchGroup: Equatable, Sendable { let groupTitle: String; let identity; let
  hits: [GlobalSearchHit] }`.
- `public struct GlobalSearchResults: Equatable, Sendable { let groups: [GlobalSearchGroup]; let
  totalMatches: Int; let tabCount: Int; var summary: String { "\(totalMatches) results — \(tabCount)
  tabs" } }`.
- `public static func run(sources: [GlobalSearchSource], query: String, caseSensitive: Bool, isRegex:
  Bool) -> GlobalSearchResults` — **REUSES `TerminalSearchController.computeMatches`** per source;
  drops sources with zero hits; `tabCount` = number of groups with ≥1 hit; preserves source order.
  Empty query → empty results. Invalid regex → empty (inherits the controller's validate-then-drop).

**New test** `Tests/SlopDeskWorkspaceCoreTests/GlobalSearchControllerTests.swift` (test-first):
- `testGroupsByTabAndCountsSummary` — 3 sources, 4 hits across 2 sources → `groups.count == 2`,
  `totalMatches == 4`, `tabCount == 2`, `summary == "4 results — 2 tabs"`.
- `testEmptySourceContributesNoGroup` — a source with `lines: []` is absent.
- `testExcerptAndHighlightRange` — excerpt is the full line; highlight range maps to the match column.
- `testCaseSensitiveAndRegexHonored` — parity with `TerminalSearchController` flags.
- `testInvalidRegexYieldsNoResultsNeverTraps` — validate-then-drop.
- `testEmptyQueryYieldsZeroResults`.

No view, no store, no surface — pure. Run: `swift test --filter GlobalSearchControllerTests`.
