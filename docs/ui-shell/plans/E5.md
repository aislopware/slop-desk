### Design divergences (state explicitly)

1. **Global Search is a non-modal overlay surface, NOT a first-class results *tab/pane*.** A `PaneKind.searchResults` tab (sidebar "TABS" → "SEARCH") was considered but would blast-radius every `switch PaneKind` site (PaneContainer, reconcile, isVideo, persistence decode, chooser, sidebar rows, titlebar) and risk silent rot for an "M" epic. Chosen: a **large, non-scrimmed `GlobalSearchView`** hosted by `OverlayCoordinator` — query field + `Aa`/`.*` toggles + grouped results + summary, dismissed with Esc, **re-openable with ⇧⌘F** (restores last results in-memory). `PaneKind.searchResults` + "SEARCH" header = **future work**.
2. **In-buffer highlight + scroll-to-match come from libghostty** (`search:`/`navigate_search:`/`scroll_to_row`); the `N of M` counter comes from `TerminalSearchController` over `scrollbackTextLines()`. These can **drift**: libghostty searches its full grid; our mirror is the `scrollbackTextLines()` snapshot at bar-open. Behaviour: counter = mirror's count; highlight = libghostty's. They agree in the common case (same buffer). Bar refreshes the mirror on open and on `Aa`/`.*` toggle.
3. **Regex semantics = ICU (`NSRegularExpression`), not Rust `regex`** — already the case in `TerminalSearchController`. `.*` labels regex mode; ICU supports lookaheads, Rust doesn't — minor, document in the bar's help/tooltip.
4. **Global Search covers terminal scrollback only.** slopdesk has no file/folder pane → no file-name/content index → terminal panes only. Non-terminal (video) panes contribute no lines. Noted in spec mapping; future work.
5. **Empty/disconnected scrollback:** a pane that never received bytes contributes `[]` → simply absent from results. Acceptable per spec mapping.

### WI-1 — Pure `GlobalSearchController` engine + tests (WorkspaceCore, headless)

**New file** `Sources/SlopDeskWorkspaceCore/Terminal/GlobalSearchController.swift`:
- `public struct GlobalSearchSource: Sendable { let paneID: PaneID; let sessionID: …; let tabID: …; let groupTitle: String; let lines: [String] }` — one per live terminal pane.
- `public struct GlobalSearchHit: Equatable, Sendable { let source identity (paneID/sessionID/tabID); let line: Int; let column: Int; let length: Int; let excerpt: String; let highlight: Range<Int> }` — `excerpt` = full matched line; `highlight` = UTF-16 column range within excerpt for the amber highlight (mirrors `find.png`).
- `public struct GlobalSearchGroup: Equatable, Sendable { let groupTitle: String; let identity; let hits: [GlobalSearchHit] }`.
- `public struct GlobalSearchResults: Equatable, Sendable { let groups: [GlobalSearchGroup]; let totalMatches: Int; let tabCount: Int; var summary: String { "\(totalMatches) results — \(tabCount) tabs" } }`.
- `public static func run(sources: [GlobalSearchSource], query: String, caseSensitive: Bool, isRegex: Bool) -> GlobalSearchResults` — **REUSES `TerminalSearchController.computeMatches`** per source; drops zero-hit sources; `tabCount` = groups with ≥1 hit; preserves source order. Empty query → empty. Invalid regex → empty (inherits validate-then-drop).

**New test** `Tests/SlopDeskWorkspaceCoreTests/GlobalSearchControllerTests.swift` (test-first):

- `testEmptySourceContributesNoGroup` — a source with `lines: []` is absent.
- `testExcerptAndHighlightRange` — excerpt is full line; highlight maps to match column.
- `testCaseSensitiveAndRegexHonored` — parity with `TerminalSearchController` flags.
- `testInvalidRegexYieldsNoResultsNeverTraps` — validate-then-drop.
