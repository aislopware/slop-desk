## Design decisions (where a local-terminal sidebar model doesn't map 1:1 onto remote)

1. **Per-pane rows vs per-tab rows.** A conventional sidebar renders one row per *tab*; our rail is V1
   "Panes" granularity (one row per visible pane of the active session — `RailRowsBuilder.rows`), 1:1 for
   the common single-pane tab. The `#N` badge is the **tab** shortcut number (`⌘1…⌘9` target = tab
   index+1), so split-tab panes share `#N`. Keep per-pane granularity (the shipped model), attach the
   tab-derived `#N` to each row.

2. **Created/Updated timestamps are EPHEMERAL (no schema bump, no migration).** "Created" = oldest-opened
   first; `session.tabs` is already insertion order (`newTab` appends) and persistence preserves array
   order → **Created sort == array order**, no timestamp needed. "Updated" needs live recency: keep a
   **runtime** `tabLastActiveAt: [TabID: Date]` mirror on the store (NOT persisted, NOT on the `Tab`
   Codable). This avoids a `Tab`/`Session` schema bump and any v11→v12 migration (no-backcompat — nothing
   to migrate). On relaunch recency resets to load order — acceptable for live-activity "Updated".

3. **By-Project key source.** A true `git rev-parse --show-toplevel` is a **host** fact (terminal is
   remote; client has no local FS). Faithful equivalent reuses E4's `gitStatus` RPC. To keep E6
   self-contained, the ordering engine takes a `projectKey: (PaneID) -> String?` **closure**; the store
   implements it preferring a cached git toplevel and **falling back to `lastKnownCwd`**. By-Project works
   immediately on cwd (WI-3); WI-7 upgrades the key to the real git toplevel (adds `repoRoot` to the E4
   payload + eager debounced fetch when grouping == byProject). If WI-7 is cut, By-Project still groups by
   cwd — never a hard dependency.

4. **Manual drag-reorder must not tear down sessions.** `moveTab` only permutes `session.tabs`; leaf set
   unchanged so `reconcileTree()` is a registry no-op (no teardown, no surface rebuild — memory rule "never
   tear down surface"). Manual sort = render rows in `session.tabs` order; drag updates that array.

5. **Badge fusion priority.** One badge per row, resolved by pure `TabBadgeResolver` with fixed precedence
   (most-urgent wins), from `progress-state.md` + `parallel-tasks.md`:
   `awaitingInput (agent needsPermission / hand)` > `error (completion .failure / agent error)` >
   `running (isShellBusy / agent working — spinner)` > `sudo (shield)` > `caffeinate (coffee)` >
   `completed/finished (completion .success — checkmark→accent dot)` > `none`. Caffeinate/sudo sit below
   active states so a running privileged command still spins; they surface when the shell is at rest,
   matching the spec's "session is active" semantics.

### WI-3 — Tab grouping/sort preference + pure `TabOrderingEngine` + `moveTab` (WorkspaceCore, headless)

**New file:** `Sources/SlopDeskWorkspaceCore/Workspace/Tabs/TabOrdering.swift`
- `public enum TabGrouping: String, Codable, Sendable, CaseIterable { case none, byProject, byDate }`
- `public enum TabSort: String, Codable, Sendable, CaseIterable { case created, updated, manual }`
- `public struct OrderedTabGroup: Equatable, Sendable { let header: String?; let tabIDs: [TabID] }`
  (`header == nil` ⇒ ungrouped flat list, no section chrome).
- `public enum TabOrderingEngine` with one pure static:
  ```
  static func groups(tabs: [Tab],
                     grouping: TabGrouping,
                     sort: TabSort,
                     projectKey: (PaneID) -> String?,
                     lastActiveAt: (TabID) -> Date?,
                     now: Date) -> [OrderedTabGroup]
  ```
  - **sort within a group:** `.created`/`.manual` ⇒ preserve incoming `tabs` order; `.updated` ⇒
    stable-sort by `lastActiveAt` descending (nil last). Ordered comparison (stable sort by key), never a
    bare `<` ternary on a possibly-equal key — ties preserve array order.
  - **grouping:** `.none` ⇒ one `OrderedTabGroup(header: nil, …)`. `.byProject` ⇒ bucket by
    `projectKey(tab.activePane)` (nil ⇒ "Other"); headers = project's last path component; groups ordered
    by first-appearance in `tabs`. `.byDate` ⇒ bucket by `lastActiveAt` into Today/Yesterday/Earlier (via
    `Calendar`; pure given `now`).
  - Pure, deterministic, no SwiftUI, no clock read (caller passes `now`) — fully headless-testable.

**New file (pure op):** `Sources/SlopDeskWorkspaceCore/Workspace/Domain/Tree/WorkspaceTreeOps+MoveTab.swift`
(or add to `WorkspaceTreeOps.swift`)
- `static func moveTab(from: Int, to: Int, in ws: TreeWorkspace) -> TreeWorkspace` — permute the active
  session's `tabs` (clamp indices; keep `activeTabIndex` on the SAME tab id after the move). Mirrors
  `selectTab`'s shape.

**Edit:** `WorkspaceStore.swift`
- Add observable state: `public internal(set) var tabGrouping: TabGrouping = .none`,
  `public internal(set) var tabSort: TabSort = .created`, runtime `var tabLastActiveAt: [TabID: Date] = [:]`.
- Public `setTabGrouping(_:)` / `setTabSort(_:)` (idempotent) that **persist** through the existing
  `PreferencesStore` seam (new `SettingsKey` entries `tabGrouping`/`tabSort`, default `none`/`created`). On
  init, hydrate both from preferences.
- `public func moveTab(from: Int, to: Int)` → `tree = WorkspaceTreeOps.moveTab(...)`; `reconcileTree()`
  (registry no-op). Switch `tabSort` to `.manual` on a manual drag.
- Stamp recency: in `selectTab(_:)` set `tabLastActiveAt[activeTabID] = Date()`; also bump on agent/command
  activity for the owning tab (in `handleAgentSignal` and the completion-badge set path, resolve the pane's
  tab id and stamp). Prune `tabLastActiveAt` to live tab ids on reconcile.
- `public func paneProjectKey(_ id: PaneID) -> String?` — prefer cached git toplevel (`paneGitToplevel[id]`,
  populated by WI-7; empty until then), else `tree.activeSession?.specs[id]?.lastKnownCwd`.
