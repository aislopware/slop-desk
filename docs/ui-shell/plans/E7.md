# E7 — Settings sections parity + iOS settings + import/export surfacing

Plan owner: implementation architect. Status: ready to implement. Order the work items top-to-bottom;
items that share a file are ordered so earlier edits land first.

## Goal (from BACKLOG)

Expand the 5-tab Settings window into the target 8-section taxonomy (General / Shell / Controls / Editor /
Appearance / Agents / Keybindings / Advanced), surface the 5 orphan `SettingsKey` toggles that have no UI,
add the new Controls/Scroll/Copy toggles that E8 will consume, ship a searchable **All Settings** list with
Reset-All / Reset-Advanced in the Advanced section, surface `WorkspaceTransfer` via
`.fileExporter`/`.fileImporter`, add an **iOS settings sheet**, and add the **On-Launch** setting (O1).
Plus the folded-in E1/E3 carry-over fixes (see `plans/E7-carryovers.md`).

Acceptance: ES-E7-1…ES-E7-5 (USER-STORIES.md) + every item in `plans/E7-carryovers.md`.

## Scope reductions (hard exclusions — do NOT build)

- **No horizontal tab-bar option** anywhere in Settings/UI. If a "tab bar position" row is ever rendered,
  show ONLY the vertical/sidebar choice — never a horizontal or disabled/coming-soon variant.
- **No SSH-host filter / pill** in any settings surface.
- **No terminal-config import engine** (Ghostty/Kitty/Alacritty parsing). The `import-export` spec
  describes importing *other terminals'* configs; **E7's import/export is the `WorkspaceTransfer` document
  only** (ES-E7-4 is explicit: "export the workspace to a file and import it back"). The four-bucket
  classification engine is OUT of E7.
- No "Set as Default Terminal" / "Install CLI" / OS-integration buttons (those are O2/O3, routed to E14).
  E7 ships ONLY the **On-Launch** general setting from `getting-started__first-launch.md`.

## Wire / iOS / float-math flags

- **WIRE protocol (golden corpus): NOT touched.** No new `SettingsKey` value is folded into
  `EnvConfig.overlay` or the `video-prefs.json` sidecar — the orphan + new toggles are all `Defaults.Keys`
  fire-time flags (the same channel as the existing `oscNotifications`/`redactSecrets` toggles), and the
  `WorkspaceTransfer` document envelope (`magic="slopdesk.workspace"`, `formatVersion=1`) is unchanged.
  The keybinding-registry carry-overs (`.closeWindow`) are default-chord pins (ours to re-scope), NOT a
  wire/golden constant — exactly as the E1 keymap fixes were. **Run `bash scripts/golden-check.sh` anyway
  to prove no drift.**
- **iOS sources (`#if os(iOS)`): TOUCHED** — WI-5 adds the iOS settings sheet entry in `WorkspaceRootView`'s
  `iosToolbar` + a cross-platform `SettingsSheet` host. iOS rots silently under `swift build`, so **run
  `bash scripts/check-ios.sh` after WI-5 and after any edit to `SettingsView.swift`** (it is the iOS host's
  body).
- **float-math / codec / controller code: NOT touched.** No FEC/packetizer/reassembler/QP/pacer change.
  (The Video tab keeps its existing optional QP/FEC steppers untouched; only relocated, not re-coded.)

## Exact reuse (existing engines — do NOT rebuild)

- `Sources/SlopDeskWorkspaceCore/Workspace/Store/SettingsKey.swift` — `SettingsKey` string namespace +
  the `Defaults.Keys` typed extension + the fire-time accessors. New toggles are declared HERE following
