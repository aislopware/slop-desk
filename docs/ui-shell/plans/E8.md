## 0. The load-bearing discovery (read this first)

**The terminal renders on the CLIENT via libghostty (`GhosttySurface` behind the `TerminalSurface` seam).** PATH 1 forwards raw VT bytes host→client→`feed()`→`ghostty_surface_write_output`, so every OSC-52 / OSC-22 / DECSET a *remote* program emits arrives in the **client's** libghostty — **no host round-trip, no wire extension** for this epic. Spec mapping notes assuming a host-side libghostty or a new `ClipboardWrite/Read` wire pair are **wrong for our architecture**; OSC-52 read/write is already wired client-side (`current-state/terminal-features.md` rows "OSC 52 clipboard read/write — done").

**The vendored libghostty fork (`ThirdParty/ghostty/.work/ghostty-src/src/config/Config.zig`) already implements almost every E8 knob as a native config key.** Verified present:

| target setting | libghostty config key | values |
|---|---|---|
| Copy on Select (I4) | `copy-on-select` | `false` / `true` / `clipboard` |
| Trim Trailing Spaces (I5) | `clipboard-trim-trailing-spaces` | bool (default `true`) |
| Clear Selection on Typing (I6) | `selection-clear-on-typing` | bool (default `true`) |
| Clear Selection on Copy (I6) | `selection-clear-on-copy` | bool (default `false`) |

So **the bulk of E8 is a config-passthrough job** through the EXISTING live-reload pipeline (`PreferencesStore.applyTerminal()` → `TerminalConfigBuilder.string(...)` → `TerminalConfigBroadcaster.publish` → `GhosttyTerminalView.applyTerminalConfig` → `ghostty_config_load_string`). The genuinely-custom client work is smaller: the paste-protection **sheet** (libghostty has the gate, embedder currently auto-approves), the OSC-52 **ask** sheet, the **Paste-as…** transforms, the **right-click-action** dispatch, **mouse-over-to-focus** across *slopdesk's own* panes (not ghostty's split tree), **OSC-22 pointer shape**, **cursor color UI + live preview**, and three honest-ceiling items (backspace-deletes-selection, scroll-past overscroll, cursor-smooth-animation) where the pinned fork exposes no API.

## 4. Approach per acceptance story + verification

### ES-E8-1 — Copy-on-Select (I4) + Trim-Trailing (I5) — *both*
Emit `copy-on-select = clipboard`/`false` and `clipboard-trim-trailing-spaces = true/false` via config passthrough (WI-2). libghostty owns the selection→pasteboard write (routed to the private SELECTION pasteboard, so copy-on-select does not clobber the system clipboard until an explicit ⌘C — the existing `slopdeskPasteboard(for:)` split). **Verify:** unit — `TerminalConfigBuilderControlsTests` pins both lines on/off. GUI — `scripts/check-macos.sh`: select text, confirm clipboard updated; toggle trim, copy a trailing-space line, confirm trimmed.

### ES-E8-2 — Clear-on-typing / Clear-on-copy + Backspace-deletes-selection (I6, I7) — *both*
Clear-on-typing/copy = `selection-clear-on-typing` / `selection-clear-on-copy` config keys (WI-2; libghostty-native). Backspace-deletes-selection (I7) = `BackspaceSelectionPolicy` + keyDown intercept gated to the prompt zone (OSC-133 via `TerminalModeTracker`, non-alt-screen). **Ceiling:** the pinned fork exposes **no programmatic set-selection / cursor-geometry API** (current-state note, `TerminalViewModel.swift:303-308`), so a *faithful* "delete the whole selection wherever it is" is not achievable. Closest faithful equivalent: when the policy says `deleteSelection` and the selection is a single prompt-line run ending at the cursor (common case), send `readSelection().count` DEL bytes; else fall back to `clearThenSingle`. **Verify:** unit — `BackspaceSelectionPolicyTests` pins the 3-way decision incl. prompt-zone + alt-screen gates. GUI/HW — Phase-3 self-verify: select a prompt run, Backspace, confirm whole run gone; repeat inside `vim` (alt-screen) → single-char passthrough.

### ES-E8-3 — Paste Protection sheet (I9) — *both*
libghostty already trips `clipboard-paste-protection`; today the embedder **auto-approves** (`confirm_read_clipboard_cb` → `confirmed: true`). Replace with: run `PasteSafetyAnalyzer`, and if it warns AND not skipped (full-screen TUI via alt-screen, OR `clipboard-paste-bracketed-safe` on + program advertised ?2004h) → present `PasteProtectionSheet`; **Paste Anyway** → `confirmed: true`, **Cancel** → libghostty cancel contract. **De-risk (do FIRST in WI-4):** the existing comment warns `confirmed: false` *re-trips the same gate → unbounded recursion → crash*; before wiring, confirm in `ghostty.h` how to *cancel* a pending clipboard request (a dedicated cancel/complete-empty call, not a re-ask). Skip the whole sheet inside a full-screen TUI. **Verify:** unit — `PasteSafetyAnalyzerTests` (multi-line, trailing-newline, `sudo `/`su `, control-char; skip on alt-screen + bracketed-safe). GUI — `check-macos.sh`: paste a multi-line `sudo` block at the shell → sheet; paste same in `vim` → no sheet.

### ES-E8-4 — Paste as… (I10) — *both*
`PasteTransform` + new `TerminalContextMenu` items + an **Edit ▸ Paste as** submenu. Selection → `surface.readSelection()` typed via `surface.text`; File-Base64 → `NSOpenPanel`→`PasteTransform.base64`; Escaped → `PasteTransform.shellEscaped`; Bracketed → `PasteTransform.bracketed`; →Composer → `model.onPasteToComposer` (client-only buffer; no wire). **Verify:** unit — `PasteTransformTests` + `TerminalContextMenuTests` (items + enablement). GUI — exercise each submenu item.

### ES-E8-5 — Scroll-past-last/first overscroll + smooth-scroll snap (I14, I15) — *GUI*
`ScrollPastPolicy` (pure) computes the overscroll anchor per mode and **returns nil on the alternate screen** (suppression). Settings (`scrollPastLastLine`/`scrollPastFirstLine`/`smoothScroll`) persist and surface. **Ceiling:** libghostty owns the viewport and exposes **no overscroll-margin / sub-row-render API** in the pinned fork (Config.zig has `scroll-to-bottom` but no `scroll-past-*` / `smooth-scroll`). So *blank-overscroll rendering* and *pixel-snap-on-gesture-end* cannot render 1:1; we land settings + policy + alt-screen gate + `mouse-scroll-multiplier`/`scroll-on-output` behaviour, and DECISIONS records the rendering as deferred pending a libghostty viewport hook. **Verify:** unit — `ScrollPastPolicyTests` (every mode + alt-screen→nil). GUI — confirm alt-screen suppression and settings round-trip; note the rendering ceiling in the verify log.

### ES-E8-6 — Mouse-over-to-focus (H6) + Right-Click-Action (H7/H8) + Hide-mouse-when-typing (H9) + OSC-22 pointer (H14) — *GUI*
- **H6:** client hover→`onRequestFocus` gated by `focusFollowsMouse` (libghostty's own `focus-follows-mouse` only covers ghostty's internal split tree; slopdesk panes are separate surfaces, so the focus relay must be ours). Short-circuit when already focused to avoid title-bar flicker.
- **H7/H8:** `rightMouseDown` branches on `RightClickAction`; `⌃`-right always menu.
- **H9:** `mouse-hide-while-typing = true/false` config makes libghostty DECIDE to hide; the hide itself is delegated to the embedder via `GHOSTTY_ACTION_MOUSE_VISIBILITY` (`Surface.zig` `hideMouse`/`showMouse`). So `action_cb` MUST also handle it → `MouseVisibilityMapping.isVisible(forRawValue:)` ({0,1}-guarded) → `NSCursor.setHiddenUntilMouseMoves(!visible)` (mirrors ghostty `setCursorVisibility`). The config line alone is INERT without this actuation.
- **H14:** `action_cb` handles `GHOSTTY_ACTION_MOUSE_SHAPE` → `PointerShapeMapping` → `NSCursor`; reset to arrow on `default`/program-exit.

**Verify:** unit — `PointerShapeMappingTests` (enum→token), `MouseVisibilityMappingTests` (visibility enum read, {0,1}-guard), `TerminalControlsTests` (right-click enum round-trip). GUI — `check-macos.sh`: hover a non-focused pane → focus moves; right-click with each action setting; type → pointer hides; run `yazi`/`btop` → pointer shape changes.

### Cursor color/opacity + live preview (H4/H5) — *GUI*
`cursor-color`/`cursor-opacity`/`cursor-text` from `TerminalPreferences` (WI-2) + `CursorPreviewView`. **Verify:** unit — builder pins the three lines. GUI — change color/opacity, confirm live preview + the real caret update (live-reload).

### Undo/redo at prompt (I18) — *both*
`PromptEditPolicy`: ⌘Z → `0x1f` (readline undo) when in the prompt zone (non-alt-screen); redo documented-omit. keyDown intercept. **Verify:** unit — `PromptEditPolicyTests`. GUI/HW — type, ⌘Z at prompt undoes; ⌘Z in `vim` passes through.

## 7. Flags

- **touchesWire:** **NO.** Everything renders in the client libghostty (OSC-52/22 arrive over the existing PATH-1 byte stream); no new message type, no version bump. `scripts/golden-check.sh` MUST stay zero-diff — the new fire-time `Defaults` keys are deliberately kept OFF the EnvConfig overlay / `video-prefs.json` sidecar (golden-safe by construction, like the E7 stubs).
- **touchesIOS:** **YES.** New mouse/NSCursor/right-click/menu code is `#if os(macOS)`; the pure engines, `TerminalControls`, `TerminalPreferences`, and the config builder compile in the iOS slice. Run `scripts/check-ios.sh` (iOS code rots silently under `swift build`).
- **touchesFloatCodec:** **NO.** No FEC/codec/controller/SIMD/hash code. `ScrollPastPolicy` uses integer `*+`/`+`/ordered clamps; `cursorOpacity`/`scrollMultiplier` are plain settings doubles (formatted, not fused — never `addingProduct`/`fma`); any min/max uses ordered comparison, not bare `</>`.
- **Hostile-input / hang-safety:** all UDP-untrusted rules N/A (client-local). Clipboard callbacks read C `request`/`location` as `!= 0`/explicit enum, never force-unwrap; `PasteTransform.base64` reads file bytes defensively. NO test instantiates `SCStream`/`VTCompression`/`VTDecompression`/Metal/`ghostty_*` — the GUI wiring is compile + code-review + `check-macos.sh` only.
