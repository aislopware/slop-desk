- **Untrusted-input rule:** the host parsers (WI-1) and bounded reads (WI-2) are validate-then-drop, no

- **Hang-safety:** no SCStream/VT*/Metal in any test. The probe's subprocess/FileHandle paths stay
  compiled-and-reviewed only (WI-2); only the PURE `String→struct` parsers get xctest (WI-1).

**New** `Tests/SlopDeskHostTests/HostMetadataProbeParsingTests.swift` (`@testable import SlopDeskHost`,

- `parseLsof`: `c<cmd>` then `n*:8080` → port 8080, procName=cmd; `n127.0.0.1:80` → 80; `n[::1]:443` →
  443 (port-after-LAST-colon for IPv6); malformed `nfoo` (no colon) → SKIPPED; `n*:notaport` → SKIPPED

### WI-2 — Bound opaque reads at the source (carry-over 4) · `#if os(macOS)` · compiled+reviewed

**Why.** `readAgentSession` does `Data(contentsOf:)` on the WHOLE session file and `gitDiff` drains the full
`git diff` pipe via `readDataToEndOfFile()` before `cappedOpaque()` trims to 15 MiB — an unbounded per-request
RAM spike. The plan specified capping BEFORE reading.

**Edit** `Sources/SlopDeskHost/HostMetadataProbe.swift`:
- Add a probe-side cap `private static let maxOpaqueReadBytes = 15 * 1024 * 1024` (mirrors
  `MetadataResponseBuilder.defaultMaxOpaquePayloadBytes`; read `cap + 1` so `cappedOpaque()` still trims an
  already-bounded tail and the "was truncated" signal survives).
- `readAgentSession(id:)`: replace `try? Data(contentsOf: URL(...))` with a length-limited read —
  `FileHandle(forReadingFrom:)` + `try handle.read(upToCount: maxOpaqueReadBytes + 1)`, `defer { try?
  handle.close() }`. Keep the root-confinement + `isReadableFile` guards (validate-then-drop; `nil` on
  failure, never a trap).
- `runProcessData(_:_:)`: replace the single `readDataToEndOfFile()` with a **byte-budgeted drain loop** —
  read chunks via `availableData` (or `read(upToCount:)`), accumulate, and once it exceeds
  `maxOpaqueReadBytes + 1` `process.terminate()` + stop reading (then `waitUntilExit()`); still drains-before-wait
  so a large diff can't deadlock the pipe. Return the bounded buffer (`nil` only on spawn failure, unchanged).
  Add pure helper `static func opaqueBudgetExceeded(_ accumulated: Int) -> Bool { accumulated >
  maxOpaqueReadBytes }` so the loop predicate is unit-pinned without I/O.
- **Hang-safety:** FileHandle/Process paths stay out of xctest (compiled+reviewed). Only
  `opaqueBudgetExceeded` is unit-tested (one boundary assertion in `HostMetadataProbeParsingTests`: `cap` →
  false, `cap+1` → true).

**Edit** `docs/DECISIONS.md`: one line — "opaque host reads (`readAgentSession`/`gitDiff`) are bounded
at the SOURCE to ≤ the 15 MiB opaque cap, so `cappedOpaque()` only trims an already-bounded tail."

**Verify:** `swift build`; `swift test --filter HostMetadataProbeParsingTests` (budget boundary). No GUI.
